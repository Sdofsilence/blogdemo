# c++特性实现的设计模式
## 通义的回答
1. auto decltype 取代工厂方法
2. 可变参数模版取代visitor模式
3. funcation和lambda取代策略和命令模式
4. 智能指针取代单例模式
5. 右值引用取代享元模式

## gemmeni的回答
1. 单例模式 c++特性:静态成员延迟初始化配合delete拷贝构造和拷贝赋值运算符。或者std::call_once配合once_flag。
2. 工厂方法模式：c++特性：可以使用函数对象或lambda来注册和调用对象的创建逻辑，而无需显式地创建工厂类继承体系。
3. builder模式：可以利用构造函数的链式调用来逐步构造对象，某些场景可以作为简化方案。
4. 适配器模式：c++特性：模版可以在编译时实现某些适配功能，特别时当适配不同类型但具有相似行为的接口时。
5. 装饰器模式：c++特性：模版结合函数对象来实现更灵活的装饰器。
6. 策略模式：c++特性：函数对象和lambda可以作为轻量级替代品。
7. 命令模式：c++特性：函数对象和lambda可以作为命令对象的轻量级替代品。
8. 观察者模式：qt的信号和槽机制或其他类似的库，提供了一种类型安全且灵活的观察者模式。
