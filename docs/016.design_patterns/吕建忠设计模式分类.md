# 八大设计原则(solid+3)
## solid
1. single responsibility 单一职责原则:一个类应该仅有一个引起它变化的原因。变化隐含着类的责任。
2. open/close 开放封闭原则:对拓展开放对修改关闭
3. liskov 里氏替换原则:子类必须可以替换基类(is-a)
4. interface segregation 接口隔离原则(最小):客户程序不应依赖不需要的接口方法;接口应该小而完备
5. dependency inversion 依赖倒置原则:高层模块不应该依赖于底层模块，而都依赖于抽象;抽象不应该依赖于实现，而都依赖于抽象。(依赖注入ID)

## 其他设计原则
1. least knowledge principle (最少知道原则)迪米特法则:一个类或模块对其他类或模块知道得越少越好。
2. 组合(多态的组合指针组合)优于继承(容易应对变化)
3. 封装变化点(隔离变化和不变的部分)
4. 面向接口编程(接口应该是不容易变化的)
    * 不将类型声明为具体类声明为一个抽象接口类
    * 客户程序无需获知具体的类，只需知道对象所具有的接口
    * 减少系统各部分的依赖关系(代码表现：局部变量、方法的参数或者对静态方法的调用)(高内聚、低耦合)

# 从封装变化的角度对模式分类
* 带...的是不常用的模式

## 组件协作
* template method(框架固定的算法但是部分支持变化)
* strategy 同一个算法的不同实现需要支持拓展和运行时切换
* observer/Event 订阅-发布 消息通知机制

## 单一职责原则
* decorator(增加额外的功能且防止子类膨胀):具体装饰接口继承需要装饰的类且组合一个需要装饰的类的对象。
* bridge(两个维度的变化)原始的定义是抽象和实现的解耦(在抽象中包含一个实现的接口的指针)不适合大于两个维度的变化

## 对象创建
* factory method(一类产品的拓展：轮子(拓展不同品牌))(对象创建依赖工厂类具体对象的注入)
* abstract factory(多类相关产品的拓展：汽车(含发动机、轮子、车框、内饰)(不同的组成不同的品牌))
* prototype (获取初始化复杂或者中间对象的clone())其实就是实现一个cloneable()接口
* builder...(分步骤(稳定)创建、但是各个部分变化的复杂对象的创建)

## 对象性能
* singleton
* flyweight 享元模式(大量的细粒度对象) (对象池一般用map实现)

## 接口隔离
* facade 是一种架构模式(客户程序不依赖于子系统中的接口，而是在所有接口上面再封装一层交互接口，隔离子系统内部的变化)为子系统中的一组接口提供一个稳定的高层接口，这个接口使得子系统更加容易使用(复用)。
* proxy(为其他对象提供一种代理以控制(隔离)对这个对象的访问)结构类似于装饰器模式，继承接口，包含对象的指针。分布式大量使用。
* adapter 继承要使用的接口包含具体被适配的子类的对象指针
* mediator...(路由器或者转发器:两个基类mediator和colleage互相包含指针，使得所有的colleage子类都可以通过mediator将消息转发到所有colleage子类，但是colleage子类之间不直接依赖)

## 状态变化
* state 对象的状态决定行为，如何运行时动态的改变状态以改变对象行为(解耦状态和行为的耦合):将所有和特定状态相关的行为都放入一个state的子类对象中，在对象中包含一个state的指针。
* memento备忘录... 把对象的状态抽象成一个备忘录，对象包含备忘录指针，备忘录只提供getset具体值的方法和初始化方法，外部可以getmemento，后面可以setmemento从备忘录中恢复。可用对象序列化方式实现。

## 数据结构
* composite 树型结构，目的是统一单对象和嵌套对象的一致性
* chain of responsibility 所有的处理对象都实现canResolve 和 process 包含next*指向下一个处理对象，需要一个默认行为。
* iterator...实现算法和底层数据结构分离的遍历

## 行为变化
* command...(用栈或队列配合对象化方法实现命令记录撤销组合)
* visitor...(实现对象上的不同的算法:例如编译器中ast中的：错误处理、类型识别...可拓展)**缺点**对象本身的子类类层次的数量不变(依赖具体的对象类型)

## 领域问题(特定规则内的问题（例如四则运算）)
* interpreter...设计一个语言的文法和表示，并实现它的解释器，解释器依据具体的表示内容解释语言中的句子并返回结果。可以理解为实现编译原理中自己的语言来解决一个特定规则的问题。
