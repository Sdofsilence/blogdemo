# wait()虚假唤醒 和 无线等待问题
* 在并发编程中，条件变量（condition variable）的正确使用需要特别注意两个关键问题：虚假唤醒（spurious wakeup） 和 无限等待（infinite waiting）。

# 虚假唤醒
* 现象：线程从 wait() 中唤醒，但关联的条件并未真正满足（即没有收到 notify 信号）。
* 原因：1.操作系统调度优化（允许假唤醒减少上下文切换） 2.硬件中断影响 3.C++ 标准明确允许这种行为（[ISO/IEC 14882]）
* 解决办法：
    * 1.始终在循环中检查条件 + 使用布尔标志作为真实条件判断
        ```
        std::unique_lock<std::mutex> lock(mtx);
        // 使用while循环检查真实条件
        while (!data_ready) {
            cv.wait(lock);
        }
        // 此时条件真正满足
        process_data();
        ```
    * 2.C++11 可直接用带谓词的 wait：
        ```
        cv.wait(lock, []{ return data_ready; }); // 自动处理循环检查
        ```

# 无限等待（Infinite Waiting）
* 现象：线程永久阻塞在 wait() 中，即使条件已满足。
* 解决办法：超时保护
    ```
    if (cv.wait_for(lock, 100ms, []{ return ready; })) {
        // 条件满足
    } else {
        // 超时处理
    }
    ```
