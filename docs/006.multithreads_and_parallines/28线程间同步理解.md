# 线程间同步的理解
> 线程间同步可归结为防止数据竞赛（通过建立 happens-before 关系）和定义哪些副作用在什么条件下可见。

# 线程间同步的核心目标可以归结为两个主要方面：

防止数据竞争（Data Race）：通过建立 happens-before 关系来确保操作之间的顺序性和可见性。
定义副作用的可见性条件：明确哪些写操作（副作用）在什么条件下对其他线程是可见的。

* 一、防止数据竞争
    * 为了避免数据竞争，必须使用同步机制来建立 happens-before 关系。这种关系确保了：写操作和读操作之间有明确的顺序。

    * 建立 happens-before 关系的方式包括：
        * 原子操作与内存顺序：使用带有适当内存顺序参数（如 memory_order_release 和 memory_order_acquire）的原子操作来建立同步点。
        * 互斥锁（Mutex）：使用互斥量来保护共享数据，确保在同一时间只有一个线程可以访问该数据。
        * 条件变量（Condition Variable）：用于线程间的通信，确保某些操作在特定条件下发生。
        * 读写锁（Read-Write Lock）：允许多个读线程同时访问，但写操作独占访问。

    * 示例：使用原子操作避免数据竞争
        ```
        #include <atomic>
        #include <thread>
        std::atomic<int> data(0);
        std::atomic<bool> ready(false);
        void writer() {
            data.store(42, std::memory_order_relaxed); // 写入数据
            ready.store(true, std::memory_order_release); // 发布数据
        }
        void reader() {
            while (!ready.load(std::memory_order_acquire)); // 等待数据就绪
            int value = data.load(std::memory_order_relaxed); // 读取数据
        }
        ```

        * 在这个例子中：ready.store(true, std::memory_order_release) 创建了一个 release 操作。ready.load(std::memory_order_acquire) 创建了一个 acquire 操作。当 reader 的 acquire 操作看到 writer 的 release 操作的结果时，它们之间形成了一个 synchronizes-with 关系，从而建立了 happens-before 关系。这意味着 reader 可以安全地读取到 data 的值 42，而不会有数据竞争。

    * 二、定义副作用的可见性条件
        * 除了防止数据竞争外，另一个重要方面是定义哪些写操作（副作用）在什么条件下对其他线程是可见的。这涉及到理解 C++ 内存模型中的 修改顺序（Modification Order） 和 释放序列（Release Sequence）。
        * 修改顺序
            * 对于任何给定的原子变量，所有的写操作（副作用）按照一个全局一致的顺序发生，称为 修改顺序。所有线程都遵循这个顺序来观察这些写操作。
        * 释放序列
            * 释放序列指的是从某个线程执行的 release 操作开始，后续的一系列操作（包括该线程自身的写操作和其他线程的 RMW 操作），这些操作构成了一个释放序列。如果另一个线程通过 acquire 操作看到了这个 release 操作的结果，那么它也能看到释放序列中的所有写操作。
        * 示例：定义副作用的可见性
            ```
            #include <atomic>
            #include <thread>
            #include <iostream>
            std::atomic<int> flag(0);
            int shared_data = 0;
            void thread_writer() {
                shared_data = 42; // 副作用：写入共享数据
                flag.store(1, std::memory_order_release); // 发布标志
            }
            void thread_reader() {
                while (flag.load(std::memory_order_acquire) != 1); // 等待标志
                std::cout << "shared_data = " << shared_data << std::endl; // 读取共享数据
            }
            int main() {
                std::thread t1(thread_writer);
                std::thread t2(thread_reader);
                t1.join();
                t2.join();
            }
            ```
        * 在这个例子中：shared_data = 42 是一个副作用。flag.store(1, std::memory_order_release) 是一个 release 操作。flag.load(std::memory_order_acquire) 是一个 acquire 操作。如果 thread_reader 中的 acquire 操作读到了 flag 的值 1，则它也能看到 shared_data 的值 42，因为 shared_data = 42 在 release 操作之前发生，并且属于同一个释放序列。
# 总结
防止数据竞争：通过建立 happens-before 关系来确保操作之间的顺序性和可见性，通常使用原子操作、互斥锁等同步机制实现。
定义副作用的可见性条件：通过理解修改顺序和释放序列的概念，确保在满足一定条件时，写操作的副作用对其他线程是可见的。
