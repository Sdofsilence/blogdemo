* 结构化数据（sql数据库表、execel表格）

# 结构化数据读取方法
* 结构化数据有固定的格式（但是一般无须手动解析）
* HTTP头是严格的结构化数据：
  * 固定的字段名
  * 明确的字段含义
  * 规范的格式要求

* 对于解析http头(这里只论解析请求报文),可以采用有限状态自动机的方式解析（请求行、请求头、空行、请求数据）

* 对于http粘包见文件["\workslog_classify_by_category\3.network_programming\网络协议tcpip_http_https\HTTP1_1和HTTP2_0的请求拆包方式.md"](https://zhuanlan.zhihu.com/p/1893088388950758956)

# 半结构化数据读取方法

* 半结构化数据有两种解析方式：(递归解析)dom树解析 和 流式解析(sax解析)
  * ()dom树解析：通过递归解析，将半结构化数据解析为dom树，然后通过dom树进行解析(这里需要注意一点，树是逻辑上的，实际上使用的是组合设计模式，每一个node中有个记录chirens的双链表头指针（这是一个实现方式，需要查查开源实现使用的有哪些，优劣势有哪些）)
  * 流式解析：边读取边解析，根据开始标签和闭合标签设置当前的状态，当遇到结束标签时，将当前状态重置为初始状态，并返回结果，例如xml数据`<books> <book> <name>Python</name></book> </books>`,状态是`+books+book+name-name-book-books`状态为空后返回结果`'books.book.name': 'Python'`
