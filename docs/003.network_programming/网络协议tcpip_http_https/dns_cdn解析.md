标准 DNS 查询流程（本地 DNS 无缓存时）
------------------------

1. **用户发起请求**  
   用户在浏览器输入 `www.example.com`，操作系统向**本地域名服务器**（递归解析器，如 ISP 提供的 DNS 或 8.8.8.8）发起 DNS 查询。

2. **本地 DNS 发现无缓存**  
   本地 DNS 没有该域名的记录，于是它**代替用户**开始迭代查询（或递归查询，取决于配置）。

3. **查询根域名服务器（Root Server）**  
   本地 DNS 向根服务器询问 `.com` 的权威服务器地址。

4. **查询顶级域名服务器（TLD Server）**  
   根服务器返回 `.com` TLD 服务器地址；本地 DNS 再向 `.com` TLD 服务器询问 `example.com` 的权威 DNS 服务器。

5. **查询权威域名服务器（Authoritative Server）**  
   TLD 返回 `example.com` 的权威 DNS 服务器（如 ns1.example.com）；本地 DNS 向该权威服务器查询 `www.example.com` 的 A 记录（IP 地址）。

6. **获取结果并返回用户**  
   权威服务器返回 IP（如 93.184.216.34），本地 DNS 缓存该结果，并返回给用户。

查询失败的最终结果
---------

如果本地 DNS 在整个迭代查询过程中**无法从权威服务器获得有效 IP 地址**，它会：

* 向用户返回一个 **DNS 错误响应**，常见类型包括：
  
  * **NXDOMAIN**：域名不存在（权威服务器明确说没有这个记录）
  * **SERVFAIL**：服务器故障（权威 DNS 无法响应或配置错误）
  * **Timeout**：查询超时（网络不通或服务器无响应）

* 用户端表现为：
  
  * 浏览器显示：“找不到 example.com” 或 “DNS_PROBE_FINISHED_NXDOMAIN”
  * 应用程序报错：“无法解析主机名”

说明
----

* 本地 DNS 的角色是“代理查询者”，它负责完成整个解析链。
* 只有当它**最终无法从权威服务器获得答案**，才会向用户返回失败。
* 如果只是本地 DNS 自己没有缓存，**不会导致失败**——这正是 DNS 设计的正常流程。

## CDN服务（内容分发网络）

* CDN是由遍布全球的无数台服务器组成的网络。这些服务器就是“CDN节点服务器”。它们的主要任务是把网站的原站服务器上的静态资源（如图片、CSS、JavaScript、视频等）缓存到自己这里。

* 当用户请求一个网站时，如果该网站使用了CDN，用户的请求最终会被引导到离他**最近**的一个CDN节点服务器上。如果这个节点上已经有用户要的内容（缓存命中），它就直接把内容发送给用户，速度极快。如果节点上没有（缓存未命中），它才会去源站服务器拉取内容，缓存下来，再发给用户。

* 主要目的：**加速访问**，**减轻源站压力**，**提高可用性**（源站暂时宕机依赖cdn节点），**防御网络攻击**（流量被分散到了各个节点）
  
  

## CDN服务具体步骤

1. cdn是在dns的基础上，再做一次dns解析，即dns服务的最后一部权威域名服务器，不返回请求的ip。

2. **CNAME记录**：权威DNS服务器上，`www.example.com` 的记录并不是一个普通的A记录（直接指向IP），而是一个 **CNAME记录**（别名记录）。这个别名通常指向CDN服务商提供的域名，例如 `www.example.com.cdn.dnsv1.com`。

3. **再次解析**：本地DNS服务器现在需要去解析这个新的CDN域名（`www.example.com.cdn.dnsv1.com`）。这个域名的权威DNS服务器是由CDN服务商管理的，我们称之为 **GTM/MNS（全局流量管理系统）**。

4. CDN的GTM系统接收到本地DNS服务器的查询请求。它会根据一套智能策略，为这个**特定的用户**计算出**最佳的CDN节点服务器**。GTM系统将计算出的最佳CDN节点的IP地址返回给本地DNS服务器。

5. **最终响应**：本地DNS服务器终于拿到了 `www.example.com` 的真正IP地址（即最佳CDN节点的IP），并将其返回给用户的浏览器。

6. **发起内容请求**：浏览器现在知道了资源在哪里，它不再需要联系遥远的源站，而是直接向这个最佳的CDN节点服务器发起HTTP/HTTPS请求。

7. **CDN节点处理请求**：
   
   * **缓存检查**：CDN节点服务器收到请求后，会在自己的本地缓存中查找用户请求的文件（如图片、CSS文件）。
   
   * **缓存命中**：如果文件存在且未过期（缓存命中），CDN节点会直接将文件返回给用户。**至此，访问完成，速度极快。**
   
   * **缓存未命中/回源**：如果文件不存在或已过期（缓存未命中），CDN节点会代表用户向网站的**源站服务器**发起请求，去拉取这个文件。
   
   * 从源站获取文件后，CDN节点会：  
     
     * a. **缓存文件**：根据规则将文件保存在本地，以备后续用户请求。  
     * b. **返回文件**：将文件返回给用户的浏览器。
