# tcp 粘包

* 所谓的tcp粘包其实是一种不准确的说法，因为tcp是面向流的套接字，本来就是没有边界和大小的(但是会有物理层/数据链路层的mtu tcp的mss限制包的大小)
* 解决的方法：

| 方法           | 优点          | 缺点           | 适用场景             |
|:------------ |:----------- |:------------ |:---------------- |
| 长度前缀 简单可靠，高效 | 需要额外包头开销    | 二进制协议，性能要求高  |                  |
| 定长包          | 最简单，无解析开销   | 不灵活，浪费带宽     | 固定格式数据流          |
| 分隔符          | 人类可读，灵活     | 需要转义，解析复杂    | 文本协议（HTTP、Redis） |
| 状态机          | 最灵活，可处理复杂协议 | 实现复杂，容易出错    | 复杂协议（HTTP、SIP）   |
| 流水线          | 高性能，减少系统调用  | 实现复杂，需要缓冲区管理 | 高性能服务器           |
