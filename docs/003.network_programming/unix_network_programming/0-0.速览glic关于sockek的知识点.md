## 套接字类型

* int SOCKET_STREAM 它通过与特定远程套接字的连接运行，并以字节流的形式可靠地传输数据。
* int SOCKET_DGRAM 用于不可靠地发送单独寻址的数据包。
* int SOCKET_RAW 此样式提供对低级网络协议和接口的访问。普通用户程序通常不需要使用这种样式。

## 套接字地址

* 使用 socket 函数新创建的套接字没有地址。其他进程只有在给它一个地址时才能找到它进行通信。我们将此称为将地址绑定到套接字，方法是使用 bind 函数。如果其他进程要找到套接字并开始与之通信，则只需关注套接字的地址。您可以为其他套接字指定地址，但这通常毫无意义;首次从套接字发送数据或使用它启动连接时，系统会自动分配一个地址（如果您尚未指定地址）。
* 套接字地址的详细信息因您使用的命名空间而异。有关具体信息，请参阅本地命名空间或 Internet 命名空间 。
* 无论命名空间如何，您都使用相同的函数 bind 和 getsockname 来设置和检查套接字的地址。这些函数使用虚假数据类型 struct sockaddr * 来接受地址。实际上，当你把它传递给bind()时，地址存在于适合您使用的地址格式的其他数据类型的结构中。

### 地址格式

* 函数 bind 和 getsockname 使用泛型数据类型 struct sockaddr * 来表示指向套接字地址的指针。您无法有效地使用此数据类型来解释地址或构造地址;为此，您必须为套接字的命名空间使用正确的数据类型。因此，通常的做法是构造一个特定于命名空间的正确类型的地址，然后调用 bind 或 getsockname 时，将指针强制转换为结构 sockaddr * 。
* 数据类型：struct sockaddr
  * short int sa_family : 这是该地址的地址格式的代码。它标识了以下数据的格式。
  * char sa_data[14] : 这是实际的套接字地址数据，与格式相关。它的长度还取决于格式，很可能超过 14。
* 地址格式名称：
  * AF_LOCAL 这指定了与本地命名空间一起使用的地址格式。AF_UNIX 同义词，AF_FILE 同义词
  * AF_INET 这指定了与 Internet 命名空间相关的地址格式。
  * AF_INET6 这类似于 AF_INET，但指的是 IPv6 协议。
  * AF_UNSPEC 这没有指定特定的地址格式。它仅在极少数情况下使用，例如清除“连接”数据报套接字的默认目标地址。

### 设置套接字地址

* 使用 bind 函数为套接字分配地址。头文件 <sys/socket.h>。
* int bind（int socket， struct sockaddr *addr， socklen_t length）：addr 和 length 参数指定地址;地址的详细格式取决于命名空间。地址的第一部分(sa_family)始终是格式指示符，它指定命名空间，并表示地址采用该命名空间的格式。
* 成功时返回值为 0，失败时返回值为 -1。error 可能的错误码。

### 读取套接字地址

* 使用 getsockname 函数读取套接字的地址。头文件 <sys/socket.h>。
* int getsockname（int socket， struct sockaddr *addr， socklen_t * length-ptr）：返回有关套接字地址的信息，该地址位于 addr 和 length-ptr 参数。请注意， length-ptr 是一个指针;您应该将其初始化为 addr 的分配大小，并在返回时包含地址数据的实际大小。
* 成功时返回值为 0，错误时返回值为 -1。

## 本地命名空间

* 符号名称（创建套接字时需要）PF_LOCAL。本地命名空间也称为“Unix 域套接字PF_UNIX。另一个名称是文件命名空间PF_FILE，因为套接字地址通常作为文件名实现。

* 在本地命名空间中，套接字地址是文件名。您可以指定任何您想要的文件名作为套接字的地址，但您必须对包含它的目录具有写入权限。通常将这些文件放在目录中 /tmp 。
* 本地命名空间的一个特性是，名称仅在建立连接时使用；一旦连接建立，地址就不再有意义，可能不存在。
* 另一个特性是，您无法从另一台机器连接到这样的套接字——即使另一台机器共享包含套接字名称的文件系统也不行。您可以在目录列表中看到套接字，但连接到它永远不会成功。
* 在本地命名空间中关闭套接字后，您应该从文件系统中删除文件名。使用 unlink 或 remove 来完成此操作；
* 本地命名空间仅支持一种协议用于任何通信方式；它是协议编号 0。
* 数据类型：struct sockaddr_un 头文件 <sys/un.h>
  * short int sun_family 这标识套接字地址的地址族或格式。您应存储值 AF_LOCAL 以指定本地命名空间。
  * char sun_path[108] 这是要使用的文件名。

## Internet 命名空间

* 互联网名称空间的套接字地址包括以下组件：
  * 您要连接的机器的地址。
  * 该机器的端口号。

### internet 套接字地址格式

* 在互联网命名空间中，对于 IPv4 (AF_INET) 和 IPv6 (AF_INET6)，套接字地址由**主机地址**和该主机上的**端口号**组成。此外，您选择的**协议**实际上也作为地址的一部分，因为本地端口号只在特定协议内有意义。
* 互联网命名空间中表示套接字地址的数据类型在头文件 netinet/in.h 中定义。
* 数据类型：struct sockaddr_in
  * sa_family_t sin_family 套接字地址的地址族或格式。您应将值 AF_INET 存储在这个成员中。地址族以主机字节序存储。
  * struct in_addr sin_addr 这是 IPv4 地址。参见 主机地址 和 主机名 ，了解如何获取要存储在这里的值。IPv4 地址以网络字节序存储。
  * unsigned short int sin_port 这是端口号。参见互联网端口 。端口号以网络字节序存储。
* 当你调用 bind 或 getsockname 时，如果您正在使用 IPv4 互联网命名空间套接字地址，你应该指定 sizeof (struct sockaddr_in) 作为长度参数。
* 数据类型: struct sockaddr_in6：
  * sa_family_t sin6_family 这标识套接字地址的地址族或格式。您应将 AF_INET6 的值存储在此成员中。
  *struct in6_addr sin6_addr 这是主机的 IPv6 地址。
  * uint32_t sin6_flowinfo  通常，此字段为零。
  * uint32_t sin6_scope_id 通常，此字段为零。
  * uint16_t sin6_port 这是端口号。

### 主机地址

* 互联网上的每台计算机都有一个或多个互联网地址 ，这些数字可以识别该计算机在互联网上的所有计算机中的身份。用户通常将 IPv4 数字主机地址写成由点分隔的四个数字序列，如‘ 128.52.46.32 ’，IPv6 数字主机地址则写成由冒号分隔的最多八个数字序列，如‘ 5f03:1200:836f:c100::1 ’。
* 每台计算机还有一个或多个主机名 ，这些是由点分隔的单词字符串，如‘ www.gnu.org ’。
* 程序允许用户指定主机时，通常接受数字地址和主机名。为了打开连接，程序需要一个数字地址，因此必须将主机名转换为它所代表的数字地址。
