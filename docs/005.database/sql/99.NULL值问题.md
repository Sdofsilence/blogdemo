# sql中的NULL

* NULL 与任何值比较都返回NULL
* = NULL 是错的？正确写法是 IS NULL
* = NULL 是错的？正确写法是 IS NULL
  * 算术运算：任何涉及 NULL 的运算结果都是 NULL
  * 聚合函数如何处理 NULL？只有COUNT(*)处理，其他的都忽略NULL
  * GROUP BY 中的 NULL 会被当作一个独立的分组！
* SQL 的 WHERE 条件判断不是简单的 TRUE/FALSE，而是 三值逻辑：TRUE/FALSE/NULL
* 如何处理 NULL？**常用函数**
  * COALESCE(value1, value2, ...) → 返回第一个非 NULL 的值
  * IFNULL(value, default)（MySQL）或 ISNULL(value, default)（SQL Server）
  * NULLIF(value1, value2) → 如果两个值相等，返回 NULL，否则返回 value1
* NULL 在索引中的影响: 建议：如果某列经常用于查询且可能为空，考虑：使用默认值（如 0、空字符串）或使用 NOT NULL + 默认值
* NULL 在 JOIN 中的行为:NULL 不会与其他 NULL 匹配！

## 总结

* 如何处理NULL
  * ✅ 尽量使用 NOT NULL 约束 减少 NULL 带来的复杂性
  * ✅ 为可空列设置默认值 如 DEFAULT 0 或 DEFAULT ''
  * ✅ 查询时显式处理 NULL 用 COALESCE, IS NULL 等
  * ✅ 避免在主键中使用 NULL 主键必须唯一且非空
  * ✅ 外键可以为 NULL 表示"无关联"

* 核心准则
  * 比较 : = NULL ❌，必须用 IS NULL ✅
  * 运算 : 任何运算 + NULL → NULL
  * 聚合 : COUNT(column) 忽略 NULL，SUM, AVG 也忽略
  * GROUP BY : NULL 自成一组
  * ORDER BY : NULL 通常排在最后（可配置）
  * JOIN : NULL 不与其他 NULL 匹配
  * 逻辑判断 : WHERE 中 UNKNOWN 不返回行
