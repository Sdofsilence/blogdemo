# GROUP BY 对其他子句的影响

* select 中只可以出现GROUP BY 中的列和聚合函数
* 分组在WHERE之后，在HAVING和SELECT DISTINCT ORDER BY LIMIT之前
* 可以在HAVING 和 ORDER BY 中使用SELECT中定义的别名

## 结论表格

| 影响            | 如何影响                       |
| :-------------- | :----------------------------- |
| select          | 只能出现：分组列、聚合函数     |
| where vs having | where过滤行、having过滤组      |
| oder by         | 可排序分组列，也可排序聚合结果 |
| 结果行数        | “多行”变成“每组一行”       |

## 详细说明

1. SQL中的子查询：嵌套查询的深度解析
2. SELECT 子句：只能出现“分组列”或“聚合函数”
   * 无GROUP BY 整个表返回一行
   * 有GROUP BY 每个分组返回一行
3. WHERE VS HAVING : 过滤时机不同
   * WHERE 在GROUP BY 之前，能使用标量函数
   * HAVING在GROUP BY 之后，能使用聚合函数
4. ORDER BY 可以排序分组列(出现在ORDER BY 中的列名)或聚合结果
5. SELECT 中的别名可以在 ORDER BY 和 HAVING 中使用
6. 对使用函数的影响
   * 标量函数：作用于分组列或聚合函数结果
   * 聚合函数：按组计算
   * 窗口函数：可以跨组计算，但需要over()
