# sql窗口函数

* SQL 窗口函数允许对与当前行相关的一组行进行计算，而不会将结果合并成一个单一值。它们常用于执行聚合、排名和滚动总计等任务。

* OVER 子句定义了计算的“窗口”行。它可以：
  * PARTITION BY: 将数据划分为组。
  * ORDER BY: 指定每个组内行的顺序。
  * ROWS/RANGE: 窗口定义，它可以是 ROWS 或 RANGE。如果框架单元是 ROWS，则当前行和框架行的偏移量是行号；如果框架单元是 RANGE，则偏移量是行值。

## 虽然 SQL 窗口函数非常强大，但用户可能会遇到一些常见的陷阱和挑战

* 分区错误：确保正确使用 PARTITION BY 子句。如果没有定义分区，整个结果集将被视为一个单一窗口。
* 窗口内的 ORDER BY：窗口函数内的 ORDER BY 子句决定了计算的顺序。始终验证它是否与您的计算逻辑一致。
* 性能考虑：窗口函数可能计算成本较高，尤其是在大数据集上。始终确保您的窗口函数已优化，并在必要时结合适当的索引。
