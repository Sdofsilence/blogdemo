## SQL Operators（很多MYSQL不支持）

* 逻辑运算符 logical
  * AND
  * OR
  * IN
    * 结合列表 ：`SELECT column_name FROM table_name WHERE condition IN (condition_value1, condition_value2 .....);`
    * 结合子查询 ：`SELECT * FROM employee WHERE Ssn IN (SELECT Ssn FROM manager);`
  * NOT
  * LIKE ：用于 WHERE 子句中搜索列中的指定模式。当我们需要对字符串数据进行模式匹配时，它特别有用。LIKE 运算符与四个主要通配符一起使用：
    * %：表示零个或多个字符。它允许匹配字符串中的任何字符序列。
    * _：表示恰好一个字符。当您想匹配给定位置上的特定数量的字符时使用。
    * []：代表方括号内的任意单个字符。
    * -(连字符)：在方括号内指定字符范围。
    * 为了使 LIKE 运算符区分大小写，您可以在 MySQL 中使用"BINARY"关键字，在其他数据库系统中使用"COLLATE"关键字 。`SELECT * FROM products WHERE name LIKE BINARY 'apple%'`
  * BETWEEN AND:
    * 包含范围 – 结果集包含上下限。
    * 用途广泛 – 可与数字、日期和文本值一起使用。
  * ALL and ANY:用于将一个值与子查询返回的所有值进行比较。
    * `SELECT * FROM employee WHERE emp_id = ALL | ANY (SELECT emp_id FROM employee WHERE emp_city = 'Varanasi');`
  * EXISTS：用于检查子查询是否返回任何行(不为空)。
  * SOME：等价于ANY
* 集合运算
  * UNION（去重）
    * UNION ALL（不去重）
  * EXCEPT（差集）MYSQL不支持（LEFT JOIN...IS NULL,NOT IN子查询（注意返回空）,NOT EXISTS子查询）
  * INTERSECT（交集）MYSQL不支持（INNER JOIN,IN子查询,EXISTS子查询）
* IS NULL
* CASE ... END

## Joins

* mysql tutorial join<https://www.mysqltutorial.org/mysql-basics/mysql-join/>
* `JOIN ...ON` 和 `JOIN ... USING(col)`不同，ON指定的连接列只出现一次，且无表前缀；USING指定的连接列只出现一次，且无表前缀。但是on支持任何条件，USING只支持等值。且有两个特殊的不能使用，一个是NATURAL JOIN不能使用（自动推断）；另一个是CROSS JOIN没有连接条件。

* INNER JOIN(JOIN) ：**如果没有搭配on/using,那么单独的join其实是cross join**
* LEFT JOIN(LEFT OUTER JOIN)
  * 配合WHERE IS NULL选择只在左表的数据
* RIGHT JOIN(RIGHT OUTER JOIN)
  * 配合WHERE IS NULL选择只在右表的数据
* FULL JOIN（LEFT JOIN 和 RIGHT JOIN的组合mysql不支持,但是可以自己组合实现<https://www.geeksforgeeks.org/mysql/how-to-use-full-outer-join-in-mysql/>）
* NATURAL JOIN：NATURAL JOIN 是 INNER JOIN 的一种，它自动根据具有相同名称和数据类型的列将两个表连接起来。它只返回在**公共列**中值匹配的行。即使这些**公共列**在两个表中都存在，结果中只出现一次。
* CROSS JOIN（笛卡尔积，没有连接条件）假设左表有m行，右表有n行，那么结果为m*n行。

## 子查询 Subqueries

* SQL中的子查询：嵌套查询的深度解析 <https://developer.aliyun.com/article/1602456>

* 使用场景
  * 过滤： 根据另一个查询中的条件选择行。
  * 聚合： 应用 SUM、COUNT、AVG 等函数，并带有动态条件。
  * 更新： 使用其他表中的值来修改数据。
  * 删除： 根据另一个查询中的条件删除行。

* 可以与子查询一起使用的子句

  | 位置 | 示例 | 说明 |
  | :--- | :--- | :--- |
  | SELECT | (SELECT MAX(age) FROM users) | 标量子查询，返回单值 |
  | FROM | FROM (SELECT ...) | 派生表（Derived Table），必须有别名 |
  | WHERE / HAVING | WHERE id IN (SELECT ...) | 常见于 IN, EXISTS, =, >, 等 |
  | JOIN | JOIN (SELECT ...) | 子查询作为“临时表”参与连接 |
  | CTE(WITH) | WITH t AS (SELECT ...) | 公共表表达式，结构更清晰 |
  | INSERT/UPDATE/DELETE | INSERT INTO t SELECT ... | DML 中的子查询 |

  * 解释
    * SELECT 子句中：标量子查询（Scalar Subquery）返回单行单列（单个值）（返回 0 或 1 行，否则报错）
    * FROM 子句中：派生表（Derived Table）：可以 GROUP BY、JOIN、聚合外层查询可以进一步过滤、排序
    * WHERE / HAVING 子句中：最常见用法
      * 类型 1：标量子查询（=, >, <）类型 要求：子查询返回单值。
      * 类型 2：行子查询（Row Subquery）返回一行多列，用于比较复合条件。
      * 类型 3：表子查询（IN, ANY, ALL）返回多行单列，常用于 IN。
      * 类型 4：EXISTS 子查询（**相关子查询**）检查是否存在满足条件的记录，性能好。
    * JOIN 子句中：子查询作为连接表；子查询必须有别名，可参与 JOIN。
    * CTE（WITH 子句）：结构化子查询 （可读性强、可复用、可递归）
    * INSERT / UPDATE / DELETE 中的子查询
  * 建议
    * 优先使用 CTE：比嵌套子查询更清晰
    * 避免过深嵌套：超过 2 层就考虑用 CTE
    * EXISTS vs IN：
      * EXISTS 更适合大表（短路求值）
      * IN 适合小结果集
    * FROM 中的子查询必须有别名
    * 标量子查询确保返回单行，否则可能报错
    * 能返回多行吗？ 在 IN, EXISTS, FROM 中可以
    * 相关子查询可能慢，优先用 EXISTS 或 JOIN 优化

* 子查询的类型
* 按目的分类
  * 标量子查询：返回单个值的子查询。
  * 列表子查询：返回一组值的子查询。
  * 行子查询：返回单行多列结果的子查询。
  * 表子查询：返回多行多列结果的子查询。
* 按形式分类
  * 嵌套子查询Nested subquery 和 独立嵌套子查询
    * 独立嵌套子查询：在独立嵌套查询中，内部查询的执行与外部查询无关。内部查询首先运行，其结果直接被外部查询使用。IN、NOT IN、ANY 和 ALL 等运算符通常与独立嵌套查询一起使用。
    * 相关嵌套子查询：在相关嵌套查询中，内部查询依赖于外部查询的执行。对于外部查询处理的每一行，都会执行内部查询。**这意味着内部查询引用了外部查询的列**。相关查询中经常使用 EXISTS 关键字。
    * 嵌套查询常见运算符
      * IN 运算符：IN 运算符用于检查列值是否与子查询返回的值列表中的任何值匹配。此运算符通过避免需要多个 OR 条件来简化查询。
      * NOT IN 运算符：NOT IN 运算符根据子查询中的一组值排除行。它特别适用于过滤掉不需要的结果。此运算符有助于识别不匹配子查询中定义的条件的记录。
      * ANY 运算符：它将一个值与子查询返回的任何值进行比较。如果至少有一个比较为真，则条件得到满足。
      * ALL 运算符：它与子查询返回的所有值进行比较。只有当对每个值都为真时，条件才成立。
      * EXISTS 运算符：EXISTS 条件主要用于检查由子查询返回的存在的行。SQL 中的 EXISTS 条件用于测试相关子查询是否返回任何结果。如果子查询返回至少一行，EXISTS 条件将评估为 TRUE；否则，评估为 FALSE。
  * 相关子查询Correlated subquery 和 非相关子查询
    * 相关子查询：一个依赖子查询：**它引用外部查询的列**。依赖于外部查询的值，对于外部查询的每一行执行一次，因此在大型数据集中会变慢。
      * 使用场景：
        * SELECT WHERE \<表达式\> operator 子查询
        * UPDATE SET \<表达式\> = 子查询 WHERE
        * DELETE WHERE \<表达式\> operator 子查询
        * EXISTS/NOT EXISTS : SELECT FROM WHERE EXISTS 子查询
    * 嵌套（非相关）子查询：在外层查询之前执行一次。与外层查询无关。通常对大数据集更有效。
    * 性能考量
      * 相关子查询由于重复执行，在大数据集上可能会很慢。
      * 尽可能使用 JOIN 以提高性能。
      * 确保在用于过滤的列上建立适当的索引。
