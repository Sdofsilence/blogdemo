## 基础

* 介绍
  * SQL 系统的关键组件
    * 数据库 ：数据库是有组织的数据集合。它将数据组织成表 ，类似于具有行（记录）和列（字段）的电子表格。
    * 表 ：每个表都强制执行其列之间的规则和关系，以确保数据完整性。
    * 索引 ：索引通过允许数据库快速定位数据而无需扫描整个表来加速查询。
    * 视图 ：视图是一个虚拟表，基本上是一个可以像表一样查询的保存的 SELECT 语句。
    * 存储过程 ：这些是在数据库中预先编写的 SQL 脚本。它们可以接收输入，运行复杂逻辑并返回结果，从而提高性能、可重用性和安全性。
    * 事务 ：事务将多个 SQL 操作组合成一个单一单元。它确保所有更改都成功应用或都不应用，从而保护数据完整性（ACID 属性）
    * 安全和权限 ：SQL 包括限制访问的工具，让数据库管理员可以分配谁可以做什么，无论是访问表、执行存储过程还是更改结构。
    * 连接： 连接根据查询相关数据集所需的关系将多个表中的数据结合起来。
  * 基本规则
    * **以分号结束**(;)：每个 SQL 语句都必须以分号结束才能正确执行。
    * 不区分大小写 ：SQL 关键字（例如，SELECT，INSERT）不区分大小写。然而，表名或列名可能根据数据库管理系统区分大小写。
    * 允许空白 ：查询可以跨越多行，但请在关键字和名称之间使用空格。
    * 保留字 ：避免使用 SQL 关键字作为名称。如果需要，请用引号（" "）或反引号（`）括起来。
    * 注释
      * 单行注释：-- 注释
      * 多行注释：/\*注释\*/
    * 数据约束 ：使用 NOT NULL、UNIQUE、PRIMARY KEY 等，以确保数据准确性。
    * 字符串值 ：将字符串用单引号（'text'）括起来。
    * 命名规则：
      * 字母开头
      * 最多30个字符
      * 只使用字母、数字和下划线（_）
* SQL 命令分类
  * DDL ：数据定义语言
    * CREATE ：创建数据库、表、索引
    * ALTER ：修改数据库、表、索引
    * DROP ：删除数据库、表、索引
    * TRUNCATE ：清空表
    * RENAME ：重命名数据库、表、索引
    * COMMENT ：添加注释
  * DML ：数据操作语言
    * INSERT ：插入数据
    * UPDATE ：更新数据
    * DELETE ：删除数据
  * DQL ：数据查询语言
    * SELECT ：查询数据(以下是子句)
      * FROM ：从哪张表查询数据
      * WHERE ：查询条件
      * GROUP BY ：分组
      * HAVING ：分组条件
      * DISTINCT ：去重
      * ORDER BY ：排序
      * LIMIT ：限制条数
  * DCL ：数据控制语言
    * GRANT ：授权
    * REVOKE ：撤销授权
  * TCL ：事务控制语言
    * BEGIN TRANSACTION ：开始事务
    * COMMIT ：提交事务
    * ROLLBACK ：回滚事务
    * SAVEPOINT ：保存点
* 数据类型(依赖具体的数据库)
  * numeric ：数字类型
    * INT ：整数(BIGINT, INT, SMALLINT, TINYINT)
    * DECIMAL/NUMERIC(P,S) ：定点数
    * FLOAT ：浮点数
    * REAL ：单精度浮点数(小很多)
  * CHARACTER AND STRING ：字符类型
    * CHAR(N) ：定长字符(最大8000Bytes)
    * VARCHAR(N) ：变长字符(max:8000Bytes)
    * TEXT ：文本(最大:2GBytes/2,127,483,647 Bytes)
  * Unicode CHARACTER AND STRING ：字符类型
    * NCHAR(N) ：定长字符(最大:4000Chars)
    * NVARCHAR(N) ：变长字符(max:4000Chars)
  * 日期和时间类型
    * DATE ：日期(YYYY-MM-DD)
    * TIME ：时间(HH:MM:SS)
    * DATETIME ：日期和时间(YYYY-MM-DD HH:MM:SS)
  * 二进制数据类型
    * BINARY(N) ：定长二进制(最大:8000Bytes)
    * BLOB ：变长二进制(max:8000Bytes)
  * BOOLEAN
    * BOOLEAN ：布尔值(TRUE/FALSE)
  * SPECIAL ：特殊数据类型
    * UUID : 全局唯一标识符
    * JSON : JSON数据类型
    * XML : XML数据类型
    * AON : 音频数据类型
