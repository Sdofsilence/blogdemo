# 特殊成员函数的默认行为

* 隐式声明的默认构造函数  
  * 无用户声明的构造函数：编译器始终声明一个inline public的成员默认构造函数
  * 存在用户声明的构造函数，用户可以用default向编译器申请自动生成隐式声明的默认构造函数。
* 隐式定义的默认构造函数  
  * 隐式声明的默认构造函数 +显示预置的没被定义为弃置的默认构造函数 ，编译器自动定义它。
  * 存在用户定义的构造函数，用户可以用default向编译器申请自动生成隐式定义的默认构造函数。
* 弃置的默认构造函数
  * T隐式声明的 + 显示预置的 函数不被定义：
    * 1.T是联合体，所有成员类型都是const限定的类型或数组类型。
    * 2.T是非联合体的类，但是有成员1
    * 3.T有一个引用类型，且不带默认初始化器的非静态数据成员。
    * 4.T有一个const限定的，不可cosnt默认构造的类型，且不带默认初始化器的非变体静态数据成员。
    * 5.T有一个类类型M（或多维数组）的潜在构造的子对象，且
      * 子对象不带默认初始化的非静态成员。
      * 子对象是某个联合体中的变体对象，联合体的其他所有非静态数据成员不带默认初始化式。
      * M的默认构造函数重载决议：
        * 没有候选
        * 子对象是变体成员时，选择了非平凡的函数。
  * 不存在用户申明的构造函数时，可以用delete禁止编译器自动生成隐式定义的默认构造函数。

* **平凡的**默认构造函数
  * 构造函数非用户提供(`隐式定义` 或 `显示预置`（首个声明）)
  * 无虚函数
  * 无虚基类
  * 无拥有默认初始化器的非静态成员。
  * 直接基类都拥有平凡默认构造函数。
  * 每个类类型的非静态数据成员都拥有平凡默认构造函数。

* **合格的**默认构造函数
  * 没有被弃置的默认构造函数是合格的。
  * `合格的默认构造函数的平凡性确定了该类是否是隐式生存期类型，是否是平凡类型`。
# 类的性质

* **由用户提供**：一个函数`由用户声明`且`没有`在它的首个声明`被显式预置default或显式弃置delete`，那么它由用户提供。

* `平凡的通常是合格的，而合格的不一定是平凡的。`

## **隐式生存期类**
* 析构函数不由**用户提供**的聚合体
* 至少有一个平凡且合格的`构造函数` 和 一个平凡且未被弃置的`析构函数`。

## **可平凡复制类**copyable class
* 至少有一个**合格的** `复制构造函数`+ `复制赋值运算符`，`移动构造函数`+`移动赋值运算符`，且每个合格的函数都是**平凡的**。
* 有一个未被弃置的平凡析构函数。
简而言之：5个平凡的特殊函数(构造函数不要求)

## 1平凡类
* 是平凡可复制类
* 有一个或多个 **合格的默认构造函数**，且都是**平凡的**。
简而言之：6个平凡的特殊函数(构造函数也要求)

## 2标准布局类
* 1`非静态数据成员`都不是 `非标准布局类类型或其数组`，不是`引用类型`，都具有`相同的可访问性`
* 2没有`虚函数`和`虚基类`
* 3继承层级中`仅有**一个类**`具`有非静态数据成员`，且**该类**`的所有基类的类型`均不同于`第一个非静态数据成员`。
* 4`基类`都是标准布局

## 3POD类
* **平凡类** + **标准布局类**
* 无非POD类类型的非静态数据成员
