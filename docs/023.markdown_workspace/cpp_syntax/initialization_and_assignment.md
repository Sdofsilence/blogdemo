## 1. 初始化(Initialization)
* `变量的初始化`在构造时提供其初值。
* 初值可以由声明符 或 new 表达式的初始化器部分提供。函数调用也会发生：形参、返回值 也会被初始化。
* 对于每个声明符，`初始化器必须是下列之一`：  
  (1) = 表达式  
  (2) = {} ; = { 初始化列表 } ； = { 指派初始化器列表 }  
  (3) ( 表达式列表 ) ; ( 初始化器列表 )  
  (4) {} ; { 初始化器列表 } ; { 指派初始化器列表 }  
  1） 复制初始化语法。  
  2） 列表初始化语法(c11)。聚合初始化语法(c11前)。  
  3） 直接初始化语法。  
  4） 列表初始化语法。  
  表达式[1]：（1.不带括号的，2.逗号表达式除外的）任意表达式。  
  表达式列表：表达式[1] 的逗号分隔列表。  
  初始化器列表：初始化器子句[2] 的逗号分隔列表。  
  指派初始化器列表：指派初始化器子句[2] 的逗号分隔列表。
* 初始化器子句[2]  
  (1) 表达式  
  (2) {}  
  (3) { 初始化器列表 }  
  (4) { 指派初始化器列表 }  
语法(2)(4)统称花括号包围的初始化器列表。

* 初始化语义  
如果`没有`为对象指定初始化器，对象会 `默认初始化`。如果没有为对象指定初始化器，程序非良构。  
如果为对象指定的`初始化器是 ()`（由于语法限制不能在声明符中出现），那么该对象会`值初始化`。如果为引用指定的初始化器是 ()，那么程序非良构。  
初始化器语义定义：
  * 初始化引用，另见。
  * 初始化对象。类型T:
    * 使用语法(1)的初始化器，对象会 `复制初始化`。
    * 使用语法(2)(4)的初始化器，对象会 `列表初始化`。
    * 使用语法(3)的初始化器，对象会 `直接初始化`。
```cpp
#include <string>
 
std::string s1;           // 默认初始化
std::string s2();         // 不是初始化！
                          // 实际上声明了没有形参并且返回 std::string 的函数 “s2”
std::string s3 = "hello"; // 复制初始化
std::string s4("hello");  // 直接初始化
std::string s5{'a'};      // 列表初始化（C++11 起）
 
char a[3] = {'a', 'b'}; // 聚合初始化（C++11 起是列表初始化的一部分）
char& c = a[0];         // 引用初始化
```

### 1.1 直接初始化(Direct initialization)
以一组明确的`构造函数`实参对对象进行初始化。
* 语法  
  1）T obj(实参);  
  2) T obj{实参}；  
  3) T(其他对象)；T(实参1,实参2...);  
  4) static_cast<T>(其他对象)；  
  5) new T(实参列表)  
  6) 类::类：成员(实参列表){};  
  7) [实参](){};  
  (1) 表达式或花括号初始化列表 的**非空**带括号列表初始化。  
  (2) 花括号环绕的单个初始化器初始化一个`非类类型对象`。（对于类类型和其他使用花括号初始化列表的初始化,见列表初始化）。(花括号唯一)  
  (3) 函数式类型转换 或 以带括号的表达式列表初始化纯右值临时量。  
  (4) static_cast 临时量。  
  (5) 非空初始化器 new 表达式 初始化动态存储期对象。  
  (6) 构造函数 初始化器列表 初始化基类 和 非静态成员。  
  (7) lamda 按复制捕获的变量初始化闭包对象的成员。  

* 效果  
  * T 是数组(c++20前(非良构)/c++20起(聚合初始化数组，值初始化不带初始化器的元素，可窄化转换))
  * `T是类类型`
    * 1.同类型纯右值：(c++17起)初始化器是 `纯右值表达式`，且类型相同为T，则`用`初始化器`表达式`自身，`而非`从它实质化的`临时量`(即不在内存实例化临时量，不要求、不调用相应构造函数)，初始化目标对象。(C++17 前，编译器可以在此情况下消除源自纯右值的构造，但适合的构造函数(复制/移动)仍必须可访问:见复制消除)
    * 2. 构造函数重载决议，调用构造函数初始化对象。
    * 否则如果，T是聚合类，按 `聚合初始化`，但是允许窄化转换，不允许使用指派初始化器，不延长临时量的生存期，不进行花括号消除，并`值初始化`任何无初始化器的元素。(c++20起)
  * 否则，T是`非类`，但是源类型是类类型(可转换)，由源类型及其基类的`转换函数`，将初始化器表达式转换为所初始化的对象。
  * 否则，T是bool源类型是std::nullptr_t,则被初始化对象的值为false。
  * 否则，必要时`标准转换`，转换 其他对象 为T，所初始化对象的值为该值。

* **注解**
  **直接初始比复制初始化更宽松：复制初始化仅考虑非显式的构造函数和非显式的用户定义转换函数，而直接初始化考虑所有构造函数和所有用户定义转换函数。**  
  在使用直接初始化语法 (1)（带圆括号）的变量声明和函数声明之间有歧义的情况下，编译器始终选择函数声明。此消歧义规则有时是反直觉的，并且已被称为最烦人的分析。

### 1.2 复制初始化(Default initialization)
从另一个对象初始化对象。
* 语法  
  1） T obj = other_obj;  
  2） T obj = {other_obj};  
  3） f(other_obj);  
  4） return other_obj;  
  5） throw obj; catch(T obj);  
  6） T arr[N] = { 其他序列 };  

* 解释  
  * 1.T具名变量，跟以 等号后随一个表达式所构成的初始化器时。
  * 2.T具名变量，跟以 花括号环绕的表达式所构成的初始化器时(c++11前，c++11后归为列表初始化)。
  * 3.按值传递 实参 到函数时。
  * 4.按值返回的函数 返回时。
  * 5.按值 抛出 或 捕获 异常时。
  * 6.作为 聚合初始化 的一部分，初始化每一个提供了初始化器的元素。
* 复制初始化的效果：
  * T是类类型：(c++17起) 初始化器是纯右值表达式且类型为T(同类型)，以表达式初始化目标对象。
  * T是类类型：且其他对象(other_obj)**是**T或T的派生类， 调用T的非显式`构造函数`（即`转换构造函数`）。
  * T是类类型：且其他对象(other_obj)**不是**T或T的派生类；或T不是而其他对象是类类型；前者使用`转换构造函数`(纯右值直接初始化对象,可能不调用，但要求适合的构造函数(复制/移动)是可访问的)，后者使用`转换函数`。
  * 否则，T 和 其他对象 都不是类类型，必要时用标准转换将 其他对象的值转换成 T 的无cv限定版本。

* **转换构造函数**
  *不以说明符 explicit 声明且`可用单个参数调用` (C++11 前)的构造函数被称为转换构造函数（converting constructor） ; 隐式声明的及用户定义的`非显式(不以说明符 explicit 声明)`复制构造函数与移动构造函数也是转换构造函数。*  
  *通常说法是`转换构造函数`指定了一个从它的各实参类型（如果存在）到它的类类型的隐式转换。注意`非显式的用户定义转换函数`也指定了一个隐式转换。*

* **用户定义转换函数**
  *用户定义的转换函数在隐式转换的第二阶段被调用，第二阶段由“零个或一个转换构造函数”或“零个或一个用户定义转换函数”构成。*  
  *`如果转换函数和转换构造函数都能用于进行某个用户定义转换(前提是同时存在)`，那么`复制初始化`和`引用初始化`语境中的重载决议会`同时考虑转换函数和转换构造函数`，但在`直接初始化`语境中`只考虑转换构造函数`(不是只考虑，而是优先级考虑，因为本文 ###直接初始化 **注解** 说会考虑)。*

* 注解  
  复制初始化不如直接初始化宽容：explicit 构造函数不是转换构造函数，也不会被复制初始化考虑。  
  ```
  struct Exp { explicit Exp(const char*) {} }; // 不能从 const char* 转换
  Exp e1("abc");  // OK
  Exp e2 = "abc"; // 错误，复制初始化不考虑 explicit 构造函数
   
  struct Imp { Imp(const char*) {} }; // 可以从 const char* 转换
  Imp i1("abc");  // OK
  Imp i2 = "abc"; // OK
  ```
  另外，复制初始化中的隐式转换必须从初始化器直接生成 T，然而直接初始化则存在比如期待一个从初始化器到 T 的某个构造函数实参的隐式转换的情况。
  ```
  struct S { S(std::string) {} }; // 可以从 std::string 转换
  S s("abc");   // OK：从 const char[4] 转换到 std::string
  S s = "abc";  // 错误：没有从 const char[4] 到 S 的转换
  S s = "abc"s; // OK：从 std::string 转换到 S
  ```
  如果 其他对象 是右值表达式，那么重载决议会选择移动构造函数并在复制初始化期间调用它。这仍然被当做复制初始化；没有此种特殊情况的专门术语（如“移动初始化”）。  
  隐式转换以复制初始化的方式定义：如果能以表达式 E 复制初始化 T 类型的对象，那么 E 就可以隐式转换到 T。  
  具名变量的复制初始化中的等号 = 与赋值运算符无关。重载的赋值运算符对复制初始化没有影响。

### 1.3 列表初始化(Direct initialization)
从`花括号初始化列表` 列表初始化对象。
* 语法  
  直接列表初始化  
  (1) T obj{ 实参... };
  (2) T{ 实参... };
  (3) new T{ 实参... }；
  (4) class{ T member{ 实参... };}
  (5) class::class_name() : member{ 实参... } { ... };
  复制列表初始化(隐式的值传递都是带={}的，如函数参数，返回值，[]下标运算符， 函数式转型或临时对象的构造函数参数)  
  (6) T obj = { 实参... };  
  (7) func( {实参...} ) ;  
  (8) return { 实参... };  
  (9) obj[ {实参...} ]  
  (10) obj = { 实参... };  
  (11) U( {实参...} )  
  (12) class{ T member = { 实参... };};  
  直接列表初始化(无=号)：  
  * 1.以花括号初始化列表，初始化`具名变量`; (可为空表达式或嵌套花括号初始化列表的列表)  
  * 2.以花括号初始化列表初始化`无名临时量`。  
  * 3.`new 表达式` 初始化动态存储期对象，初始化器是花括号初始化列表。  
  * 4.无=号的，`数据成员初始化器`中。  
  * 5.构造函数 `成员初始化列表`中，初始化器是花括号初始化列表。  
  复制列表初始化(部分有=号)：  
  * 6. 有=号的 花括号初始化列表 初始化`具名变量`。
  * 7. `函数调用`，花括号初始化列表为 实参，以列表初始化对函数形参进行初始化。
  * 8. 以花括号初始化列表 作为返回表达式的`return语句`中,列表初始化返回对象。
  * 9. 用户定义`operator[]下标表达式`中，以列表初始化对重载运算符的形参进行初始化
  * 10. `复制表达式`中,以列表初始化对重载的运算符的形参进行初始化
  * 11. `函数式转型表达式`或其他`构造函数调用`中，其中花括号初始化列表用作构造函数实参；以`复制初始化`对构造函数的形参初始化(此例中的类型 U 不是被列表初始化的类型；但 U 的构造函数的形参是)。
  * 12. 使用=的`非静态数据成员初始化器`中。

* 解释  
* (c++20)如果花括号初始化列表 包含指派初始化器列表，并且 T 不是引用类型，那么 T 必须是聚合类。进行`聚合初始化`。
* T 是聚合类，且 花括号初始化列表拥有单个相同类型或派生类型的元素，那么从该元素初始化对象（对于`复制列表初始化为复制初始化`，对于`直接列表初始化为直接初始化`）。
* 否则，T是`字符数组`且花括号初始化列表拥有`单个`类型适当的字符串字面量元素，那么照常从字符串字面量初始化数组。  
**<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<<!!!**
* 否则，T是`聚合类型`，聚合初始化。
* 否则，如果花括号初始化列表 `为空`，且T是拥有默认构造函数的`类类型`，值初始化。
* 否则，T为`类类型`，考虑它的构造函数：
  * 1.检验所有接受 std::initializer_list 作为它的唯一实参，或如果剩余实参都具有默认值则为它的首个实参的构造函数，并通过重载决议与单个 std::initializer_list 类型的实参进行匹配。
  * 2.如果上一个阶段未产生匹配，那么 T 的所有构造函数都参与针对由花括号初始化列表的各元素所组成的实参集的重载决议，它会受到只允许非窄化转换的限制。如果这个阶段为复制列表初始化产生的最佳匹配是 explicit 构造函数，那么编译失败（注意：简单复制初始化中完全不考虑 explicit 构造函数）。
* 否则(`如果T不是类类型`),如果花括号初始化列表`只有一个元素`(且T要么不是引用类型，要么是引用类型而它所引用的类型与该元素的类型相同或者是它的基类) 那么直接初始化（直接列表初始化时）或复制初始化（复制列表初始化时）这个T，但不允许窄化转换。

* 否则，如果T是与元素类型不兼容的引用类型...
* 否则，如果花括号初始化列表 `没有任何元素`，那么`值初始化` T。

* 列表初始化 std::initializer_list  
类型是 std::initializer_list<E> 的对象从初始化器列表构造时，编译器如同生成并实质化 (C++17 起)一个类型是“包含 N 个 const E 元素的数组”的纯右值，其中 N 是初始化器列表中的元素个数；该数组被称为该初始化器列表的基底数组。  
基底数组中的每个元素都会以初始化器列表中的对应元素`复制初始化`，并且构造的 std::initializer_list<E> 的对象会指代基底数组。复制操作选择的`构造函数`或`转换函数`必须在初始化器列表的语境下可访问。  
从基底数组初始化 std::initializer_list 对象时会与绑定引用到临时量一样延续基底数组的生存期。
* 窄化转换
  * 1.浮点类型到整数类型
  * 2.从 long double 到 double 或 float 的转换，以及从 double 到 float 的转换，除非来源是常量表达式且不发生溢出(c++23前)
  * 3.整数类型到浮点类型，除非来源是能将它的值完全存储于目标类型的常量表达式
  * `整数或无作用域枚举类型` 到 `不能表示原类型所有值的整数类型`，除非
    * 1.来源是宽度（记为 w）小于自身类型（或者对于枚举类型来说是底层类型）的位域 且 目标类型可以表示宽度是 w 且符号与原类型一致的虚设扩展整数类型的所有值。
    * 2.来源是能将它的值完全存储于目标类型的常量表达式
  * 指针或成员指针类型到 bool

* 注解  
花括号初始化列表 不是表达式，因此它没有类型，即 decltype({1, 2}) 非良构。  
因为花括号初始化列表 没有类型，所以在将它用作重载函数调用的实参时，适用重载决议的特殊规则。  
聚合体直接从同类型的单元素花括号初始化列表进行复制/移动初始化，但非聚合体首先考虑 std::initializer_list 构造函数：
  ```
  struct X {}; // 聚合体
   
  struct Q     // 非聚合体
  {
      Q() = default;
      Q(Q const&) = default;
      Q(std::initializer_list<Q>) {}
  };
   
  int main()
  {
      X x;
      X x2 = X{x}; // 复制构造函数（非聚合初始化）
   
      Q q;
      Q q2 = Q{q}; // initializer_list 构造函数（非复制构造函数）
  }
  ```
* 语法10((10) obj = { 实参... };//赋值表达式)  
  `复制表达式`中,以列表初始化对重载的运算符的形参进行初始化。

  * **列表初始化中的隐式转换序列({}作实参如何初始化形参)**  
  在列表初始化中，实参是 花括号初始化器列表，但它不是表达式，所以到就重载决议而言的形参类型的隐式转换序列以下列规则决定：

    * T是聚合体，桥初始化器由`一个` 同类型 或 它的派生类组成，转换序列是派生类到形参类型的转换序列。
    * T是字符数组的引用，且初始化器列表拥有单个元素，元素是类型适当的字符串字面量，恒等转换。
    * T是std::initializer_list<X>，且存在从每个初始化器列表元素到 X 的非窄化隐式转换，转换序列是所需的最坏转换。如果 花括号初始化器列表 为空，那么转换序列是恒等转换。
    * 形参是“N个T的数组”，初始化器列表必须有 N 个或更少的元素，且所用的隐式转换序列是将列表（或空花括号对，如果 {} 小于 N）的每个元素转换到 T 所需的最坏隐式转换序列。
    * T是非聚合类类型X，那么选取X的构造函数C 以从实参初始化器列表初始化：
      * C 是非initializer_list构造函数，且初始化器列表拥有单个类型为X的元素(`单个同类型`)，隐式转换序列具有准确匹配等级。如果该初始化器列表拥有单个元素，它具有`派生自X的类型`，隐式转换序列具有转换等级。（注意它和聚合体的区别：`聚合体在考虑聚合初始化前直接从单元素初始化器列表进行初始化，而非聚合体在考虑任何其他构造函数之前先考虑 initializer_list 构造函数`）
      * 否则，隐式转换序列是以恒等转换作为第二标准转换序列的用户定义转换序列(非显示 单实参可调用构造函数 和 非显示转换函数(没有参数和返回值))。
    * T是可按照聚合初始化从初始化器列表初始化的聚合体，隐式转换序列是以恒等转换作为第二标准转换序列的用户定义转换序列。
    * T是引用，应用引用初始化规则。
    * T不是类，且初始化器列表拥有一个元素，那么隐式转换序列是将该元素转换到形参类型所要求者。
    * T不是类，且初始化器列表没有元素，那么隐式转换序列是恒等转换。

* **列表初始化的隐式转换规则概览**：
  * 1.聚合体初始化：  
    如果形参是聚合体（如结构体或联合体），且列表恰好有一个元素是该类型或其派生类，使用到该类型的转换。
  * 2.字符数组引用：  
    形参是字符数组引用，且列表仅含一个字符串字面量，使用恒等转换。
  * 3.std::initializer_list 初始化：  
    形参类型为 std::initializer_list<X>，列表中的每个元素都能非窄化隐式转换为X，使用这些转换。
    空列表初始化时，视为恒等转换。
  * 4.固定大小数组：  
    形参为固定大小数组的引用，列表元素数量不超过数组大小，使用元素到数组类型的转换。
  * 5.未知边界数组（C++20起）：  
    形参为未知边界数组的引用，转换列表每个元素到指定类型。
  * 6.非聚合类类型构造函数：  
    非基于 std::initializer_list 的构造函数，优先考虑单元素列表，特别是当元素类型与类类型相匹配或派生自类类型时。
    若无上述构造函数适用，考虑其他构造函数，优先级较低。
  * 7.聚合体初始化（非 std::initializer_list 构造函数场景）：  
    对于可聚合初始化的类型，使用用户定义转换，其中恒等转换作为第二标准。
  * 8.引用参数：  
    直接应用引用初始化规则，可能涉及到临时对象的创建。
  * 9.非类类型参数：  
    若形参不是类类型且列表只有一个元素，使用该元素到形参类型的转换。
    若列表为空且形参不是类类型，使用恒等转换。
  * 10.派生列表初始化器：  
    当形参为聚合类型且能按聚合初始化规则处理列表时，即使形参不是引用，也允许转换，以恒等转换为第二标准。

**>>>>>>>>>>>>>>>>>>>>**
* 当列表初始化涉及多元素，如 T t = {1,2,3}，主要遵循以下规则：
  * 1.std::initializer_list 初始化:  
    如果 T 是 std::initializer_list<X> 类型，且列表中的每个元素都可以通过非窄化隐式转换变为 X 类型，那么这个规则适用。例如，如果 T 是 std::initializer_list<int>，那么 {1,2,3} 可以直接用来初始化 T。
  * 2.固定大小数组:  
    如果 T 是一个固定大小的数组类型，比如 int[3] 或者引用到这种类型的数组，如 int(&)[3]，并且列表中的元素数量与数组大小相匹配，那么每个元素会被转换到数组的元素类型（这里是 int）进行初始化。
  * 3.非聚合类类型构造函数:  
    如果 T 是一个非聚合类类型，并且它有接受 std::initializer_list 的构造函数以外的构造函数，那么会尝试使用这些构造函数。特别是当列表只包含一个元素，且该元素类型与 T 或其基类兼容时，会考虑直接构造函数或转换构造函数。然而，如果没有这样的构造函数，且列表有多于一个元素，那么通常需要一个接受多个参数的构造函数来匹配列表中的元素。
  * 4.聚合体初始化:  
    如果 T 是一个聚合体（如结构体或基本的类类型，没有私有或受保护的非静态数据成员，没有用户自定义构造函数、赋值运算符或析构函数），那么列表中的元素会依次尝试初始化聚合体的成员。成员的初始化顺序遵循它们在类定义中的声明顺序。如果列表元素数量超过了成员数量，超出的元素将被忽略；如果不足，则剩余成员默认初始化。
  * 5.错误情况:  
    如果没有合适的构造函数或初始化方式来匹配多元素的列表初始化，编译器将会报告错误。
  * **综上所述**：  
    **对于 T t = {1,2,3} 这样的初始化，如果 T 不是 std::initializer_list 类型，且假设 T 不是一个聚合体，那么这将尝试查找一个接受三个参数的构造函数（或者是接受一个 std::initializer_list 的构造函数，如果列表元素类型与 T 内部处理类型兼容）。如果找不到合适的构造函数，编译将失败。**

### 1.4 默认初始化(List initialization)
在不使用初始化器构造变量时执行的初始化。
* 语法  
  (1) T obj;  
  (2) new T;  

* 默认初始化`三种情况`:  
  1）不带初始化器，创建自动、静态、线程局部存储期变量。
  2）不带初始化器的new 表达式，创建动态存储期对象，或以带new T() 的new表达式创建对象时。
  3）当构造函数初始化器列表 中未提及某个基类或静态数据成员，且调用了该构造函数时。

* 默认初始化的效果： 
  * 非pod 类类型，调用重载决议的构造函数(即默认构造函数之一)。
  * 数组，每个元素都被 默认初始化。
  * 否则 不初始化(不能默认初始化 引用和const标量)。

### 1.5 值初始化(value initialization)
以空初始化器构造对象时进行的初始化。
* 语法(匿名对象(`**1.new匿名对象)、2.成员初始化列表、3.T t{}具名对象**`)  
  (1) T() ; T{}  
  (2) new T() ; new T{}  
  (3) class::class(...) : member() { ... } ; class::class(...) : member{} { ... }  
  (4) T obj{};  
  1）`空`括号/`空`花括号对 组成的初始化式创建无名临时变量时。
  2）new 表达式以 `空`括号/`空`花括号对 组成的初始化式创建动态存储期对象时。
  3）当以 `空`括号/`空`花括号对 组成的`成员初始化`式初始化 非静态数据成员 或 基类时。
  4）当以由 `空`花括号对 组成的初始化式声明具名对象（自动、静态或线程局部）时。

**所有情况下，使用空花括号对{},且T是`聚合类型`，进行 `聚合初始化` 而非值初始化。**  
**如果 T 是没有默认构造函数但带有接受 std::initializer_list 的构造函数的`类类型`，那么进行`列表初始化`。**

**值初始化的效果：**
* 1.如果T是*无默认构造函数* 或拥有由用户声明的 *由用户提供的*或*被弃置的* 默认构造函数的类类型，进行`默认初始化`。
* 2.如果T是*拥有默认构造函数*的类类型，而默认构造函数不由用户声明 既 不由用户提供 又 没有弃置(它可以拥有隐式定义的或预置的默认构造函数的类)，那么 `零初始化` 对象；如果它拥有`非平凡的默认构造函数`，那么会 `默认初始化` 它。
* 3.如果T是数组类型，那么 `值初始化` 数组的每一个元素；
* 4.否则，`零初始化` 对象。

### 1.6 聚合初始化（c++11后归为列表初始化）
从初始化器列表初始化`聚合体`。此为列表初始化的一种形式。 (C++11 起)
* 语法  
  (1) T obj = {args...};
  (2) T obj{args...};
* **聚合体(c的结构体一定是聚合体)**
  * 数组
  * 类类型满足以下要求：
    * 1.无特定`构造函数`(c11前(无用户声明)/c++11起(无用户提供、using继承、explicit的)/c20 变动(无用户声明、继承))。*解释：函数`由用户声明`且 没有在它的首个声明被显式预置=default或显式弃置=delete，那么它由`用户提供`*。
    * 2.无私有/保护的直接 `非静态数据成员`(c17起)。
    * 3.无特定`基类`(c17前(无基类)/c17起(无虚基类、私有、保护的直接基类))；
    * 4.无`虚成员函数`。
    * 5.无`默认成员初始化器`(c++11起/c++14前)

### 1.7 零初始化
* 定义  
  将一个对象的初始值设置为零。
* 语法(零初始化在语言中无语法，下列是可能零初始化的例子)  
  (1) static T obj;
  (2) T();/t{}; ; T t = {};
  (3) charT arr[n] = "短序列"；
  1）静态或线程局部存储期不进行常量初始化的具名变量。
  2）`非类类型`变量，和被值初始化的无构造函数的类类型的成员的值初始化序列的一部分，包括提供初始化式的聚合体的值初始化。(`类类型详细见值初始化`提及的零初始化)
  3）短字符串字面量初始化`任何字符类型数组`时，零初始化`剩余部分`。

* 注解  
  如非局部初始化中所述，在进行任何其他初始化前， `零初始化`所有未被常量初始化的`静态和线程局部(C++11 起)存储期`的变量。如果`非类类型非局部变量`的定义没有初始化式，那么默认初始化不做任何事，不修改先前零初始化的结果。

零初始化的指针是其类型的空指针值，即使空指针的值并非整型的零。

### 2.2 内建的复合赋值
* 内建复合赋值运算符表达式 target-expr op new-value 新值 的行为和表达式 target-expr = target-expr op new-value 完全一致，除了目标表达式只会求值一次。
