# c++的四种类型转换关键字的区别

# 使用场景：
* static_cast       用于基本类型的转换，以及用户定义类型的转换，只要转换是类型系统允许的。
* reinterpret_cast  用于低级别的位模式转换，如指针和整数之间的转换。通过重新解释底层位模式在类型间转换。
* const_cast        用于移除或添加const和volatile限定符。
* dynamic_cast      用于有检查的运行时多态类型转换。
* bit_cast          将一个类型的对象表示重解释为另一类型的对象表示

## static_cast

* 1.隐式转换序列(标准转换+用户定义转换)+直接初始化。(3、4、5、6)
* 2.指针与指针。引用和指针的向下转换、成员指针的向上转换、空指针转换、void*到相似且可指针互相转换的转换。(1、10、11)
* 3.有作用枚举、浮点、整型之间的转换。(7、8、9)
* 4.右值引用类型。(2)

* 具体的各种可能使用场景
  * 1.引用和指针向下转型。
  * 2.转换为右值引用类型(亡值)。 
  * 3.隐式转换序列+直接初始化。
  * 4.static_cast<void>(expr),舍弃该值。
  * 5.**标准转换**(`数值转换和限定性转换`)(除左值到右值、数组到指针、函数到指针、空指针、空成员指针、函数指针或布尔转换外)，static可进行隐式转换的逆转换。
  * 6.左值到右值、数组到指针、函数到指针的显式转换。
  * 7.有作用域枚举类型转换(只可以显式转换，无作用域枚举可隐式转换)到整数和浮点类型。目标类型为bool时，原值为0结果为false，否则为true。
  * 8.整数或枚举类型(浮点)转换到任何枚举类型。
  * 9.浮点类型纯右值转换到任何其他浮点类型。
  * 10.指向某完整类 D 的成员的指针可以向上转换（upcast）到指向它的无歧义、可访问的基类 B 的成员。
  * 11.指向（可有 cv 限定的）void 的指针类型的纯右值可以转换成指向对象类型 T 的另一指针。(对齐要求、类型相似且可指针互相转换)

* 转换结果：
  * **左值**。目标类型是左值引用、函数类型的右值引用。
  * **亡值**。目标类型是对象类型的右值引用。
  * **纯右值**否则。

* **指针可以互相转换对象**
满足以下，对象a、b是指针可以互相转换的对象
  * 同一个对象。
  * 一个是联合体，另一个是该对象的 非静态数据成员
  * 一个是标准布局 类对象，另一个是对象的 首个非静态数据成员，或是对象的任何基类子对象。
  * 存在 对象c 使得a 与 c 的指针可以互相转换， 而 c 与 b 的的指针可以互相转换。

## reinterpret_cast

* 1.指针与指针之间。(4、6、7、8、9、10 :不同类型指针、函数指针、void*和前面两者、空指针、成员函数指针、成员对象指针 )
* 2.整数与指针之间。(1、2、3)
* 任意类型T1泛左值->T2引用。(5)

* 具体使用的场景
  * 1.指针转换到所有大小足够保存它的类型的整数。
  * 2.任何整数、枚举都可以转换到指针类型。
  * 3.std::nullptr_t -> 任何整数类型，如同反过来不行，需要static_cast。
  * 4.任何对象指针类型T1*，可以转到cv T2*，严格等价于static_cast<cv T2*>(static_cast<cv void*>(t1))。T2的对齐要求不比T1的更严格。只有在解引用后的值具有**类型可访问性时**，结果指针才可安全解引用。
  * 5.任何类型的泛左值可以转换到另一个类型T2的引用。结果与*reinterpret_cast<T2*>(p)的结果相同。p是类型为T1*，指向表达式代表的对象或函数的指针。结果引用只有在具有**类型可访问性时**，才可安全访问。
  * 6.任何函数指针都可以转换为指向不同函数类型的指针。只有在转换后的函数类型与原函数具有**调用兼容性**时才可以安全调用。
  * 7.一些实现中，函数指针可以转换成void*任何其他对象指针类型的，反之亦然。依据实现决定是否可以安全的调用。
  * 8.任何指针类型的空指针值可以任何其他指针类型，并产生该类型的空指针值。不能转换std::nullptr_t的任意值到其他类型，应使用static_cast。
  * 9.成员函数指针可以转换成指向不同成员函数的指针。
  * 10.指向T1的成员对象的指针可以转换成指向另一个成员对象的指针。T2的对齐要求不比T1的更严格。

* **类型可访问学性**(字节类型 + 相同类型的signed/unsigned间)
类型T_ref 与 以下类型之一相似，那么动态类型T_obj的对象对T_ref类型的泛左值具有类型可访问性。
  * char
  * unsigned char
  * std::byte
  * (signed/unsigned) T_obj
* 许多 C++ 编译器作为非标准语言扩展放松此规则，以允许通过联合体的不活跃成员的进行类型错误的访问。

* **调用兼容性**(相同类型 + 相同类型的noexcept/except间)
满足以下任意条件，类型T_call 与 类型 T_func具有调用兼容性。
  * T_call 与 T_func 是相同类型。
  * T_func可以通过函数指针转换转换(noexcept 到 except)到T_call*.(c++17起)

## dynamic_cast
* 1.目标类型空指针 + (多态)void*指针
* 2.cv限定转换 和 向上转换(隐式转换 换 static_cast也可以)
* 3.多态运行时检查转换。
* 4.构造和析构函数中转换当前构造或析构的对象。

* 具体使用场景
  * 1.添加cv 限定符。
  * 2.空指针值->目标类型空指针值。
  * 3.派生类到基类(唯一可访问基类)的指针或引用。
  * 4.多态类型派生类指针或引用到void指针，结果是指向最终派生类对象的指针。
  * 5.多态类型基类Base的指针或引用，目标类型是派生类Derived的指针或引用，会进行运行时类型检查。
    * a.检验表达式指向的最终派生类对象。如果表达式指向Derived的公开基类，且只有一个Derived类型对象从表达式指向的子对象派生，结果指向该Derived对象("向下转换")。
    * b.否则，表达式指向指向最终派生类对象的公开基类，同时最终派生类对象拥有Derived类型的无歧义公开基类，结果指向该Derived对象。("侧向转换")
    * c.否则，转换失败。用于指针，返回目标类型空指针值，用于引用，抛出std::bad_cast异常。
  * 6.在构造函数和析构函数中使用(直接或间接)dynamic_cast，且表达式指向正在构造或析构的对象时，该对象被认为是最终派生对象。如果目标类型不是到构造函数/析构函数自身的类或它的其中一个基类的指针或引用，那么行为未定义。

## const_cast
* 1.两个相似的类型T1和T2，仅cv不同，那么T1类型的纯右值可以转换到T2。结果指代原来的实体。
* 2.两个对象类型T1和T2，如果指向T1的指针可以通过cosnt<T2*>显式的转换到类型T2*,那么也可以进行以下转换：
  * a.T1类型的`左值`可以通过cosnt_cast<T2&>显式转换到T2的左值。
  * b.T1类型的`泛左值`可以通过const_cast<T2&&>显式转换到T2的亡值。
  * c.如果T1是类类型，那么T1类型的`纯右值`可以通过const_cast<T2&&>显式转换到T2的亡值。
* 3.空指针值可以转换成 目标类型的空指针值。空成员指针值可以转换到目标类型的空成员指针值。

* 注解  
函数指针和成员函数指针无法用于 const_cast。

* 移除常量性
当从类型T1转换到类型T2时，如果T2的限定性（qualifiers）比T1少，尤其是如果T2缺少const或volatile限定符，那么这种转换就被称为移除常量性。例如，从const int转换到int，或者从const volatile int转换到volatile int或int。

# 隐式转换
* 指针转换
  指向完整派生类的指针可以转换为指向(有相同cv限定的)基类的指针。结果是指向原对象内的基类子对象的指针。

* 成员指针转换
  指向基类 B 中的某类型T成员的指针纯右值，可以转换为指向它的完整派生类 D 中的相同类型T成员的指针纯右值。能以D对象解引用结果指针，而将它访问该D对象的 B 基类子对象内的成员。
  * 如果 B 是 D的 不可访问、有歧义的、虚基类、中间虚基类的基类，程序非良构。
  * 如果D 既没有 包含该原始成员，也不是 包含该原始成员的某个基类，行为未定义。
