合并区间（Merge Intervals）是算法面试中非常经典的一类题目，主要考察**排序**、**贪心算法**以及**数组操作**能力。这类问题通常出现在日程安排、资源调度、基因序列分析等实际场景中。

根据搜索到的资料和常见题库（如 LeetCode），合并区间类的题目主要可以分为以下几种核心类型：

### 1. 基础合并型 (Basic Merge)
这是最标准的题型，要求将所有重叠的区间合并成一个不重叠的区间集合。
*   **典型题目**：LeetCode 56. 合并区间 (Merge Intervals)
*   **核心逻辑**：
    1.  **排序**：首先将所有区间按照**起始位置**（左边界）从小到大排序。
    2.  **遍历合并**：初始化一个结果列表，放入第一个区间。遍历后续区间，如果当前区间的起始位置 $\le$ 结果列表中最后一个区间的结束位置，说明重叠，更新最后一个区间的结束位置为 `max(原结束位置, 当前结束位置)`；否则，将当前区间直接加入结果列表。
*   **时间复杂度**：$O(N \log N)$，主要消耗在排序上。

### 2. 插入合并型 (Insert and Merge)
在已排序且不重叠的区间列表中，插入一个新的区间，并保持列表有序且无重叠（可能需要合并）。
*   **典型题目**：LeetCode 57. 插入区间 (Insert Interval)
*   **特点**：输入数组本身已经是按起始位置排序的。
*   **解题策略**：
    *   可以将问题转化为“基础合并型”：直接将新区间加入数组，然后重新排序并执行合并逻辑（$O(N \log N)$）。
    *   **优化解法**：利用数组已排序的特性，一次遍历（$O(N)$）。将区间分为三部分处理：
        1.  完全在新区间左侧且不重叠的区间（直接加入结果）。
        2.  与新区间重叠的区间（更新新区间的左右边界，进行合并）。
        3.  完全在新区间右侧且不重叠的区间（直接加入结果）。

### 3. 区间移除/差集型 (Remove Interval / Set Difference)
给定一个有序不重叠区间列表和一个要移除的区间，返回移除后的区间列表。这本质上是求集合的差集。
*   **典型题目**：LeetCode 1288. 删除被覆盖区间 (Remove Covered Intervals) 的变体，或 LintCode 上的区间移除题。
*   **核心逻辑**：
    *   遍历原有区间，判断与待移除区间的关系：
        *   无交集：保留原区间。
        *   完全被包含：该部分被删除。
        *   部分重叠：将原区间切割，保留未重叠的部分（可能将一个区间分裂成两个）。

### 4. 统计与最大化/最小化型 (Counting & Optimization) **这个很重要**
这类题目不一定直接输出合并后的区间，而是基于合并的思想求解数量或极值。
*   **无重叠区间数量**：
    *   **典型题目**：LeetCode 435. 无重叠区间 (Non-overlapping Intervals)。
    *   **目标**：移除最少数量的区间，使得剩余区间互不重叠。
    *   **解法**：等价于求**最多**能保留多少个不重叠区间（贪心策略：优先选择结束时间最早的区间）。答案 = 总数 - 最大不重叠数量。
*   **最少箭数引爆气球**：
    *   **典型题目**：LeetCode 452. 用最少数量的箭引爆气球 (Minimum Number of Arrows to Burst Balloons)。
    *   **目标**：找到最少的点，使得每个区间内至少有一个点。
    *   **解法**：这也是合并思想的变体。按右边界排序，贪心地选择当前区间的右端点作为射击点，跳过所有被该点覆盖的区间。

### 5. 高级变种与多维扩展 **这个很重要**
*   **区间覆盖判断**：判断某个特定区间是否完全被其他区间的并集覆盖。
*   **会议室问题**：
    *   **会议室 I**：判断一个人能否参加所有会议（即是否存在重叠）。
    *   **会议室 II**：计算所需的最少会议室数量（扫描线算法或优先队列）。
*   **数据流合并**：区间是动态输入的（Data Stream），需要设计数据结构支持动态添加区间并实时查询合并后的结果（通常使用 `TreeMap` 或平衡二叉搜索树来维护有序区间）。

### 通用解题模板 (Python 示例)

针对最常见的**基础合并型**（LeetCode 56），标准解法如下：

```python
def merge(intervals):
    if not intervals:
        return []
    
    # 1. 按左边界排序
    intervals.sort(key=lambda x: x[0])
    
    merged = [intervals[0]]
    
    for current in intervals[1:]:
        last = merged[-1]
        # 2. 判断是否重叠
        if current[0] <= last[1]:
            # 重叠：合并，更新右边界为两者的最大值
            last[1] = max(last[1], current[1])
        else:
            # 不重叠：直接加入
            merged.append(current)
            
    return merged
```

### 总结关键点
1.  **排序是前提**：绝大多数区间问题第一步都是排序（通常按左边界，有时按右边界）。
2.  **贪心策略**：在遍历过程中，局部最优解（如尽早结束、尽可能合并）通常能导向全局最优解。
3.  **边界条件**：注意处理空数组、单元素数组、完全包含关系（如 `[1, 10]` 和 `[2, 5]`）以及刚好相接的情况（如 `[1, 2]` 和 `[2, 3]` 是否算重叠，视具体题目定义而定，通常算）。
