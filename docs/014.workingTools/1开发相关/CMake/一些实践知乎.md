    在网上还可以找到一些其他的好信息。以下是一些资源：
    
    * [官方帮助](https://cmake.org/cmake/help/latest/) ：真正令人惊叹的文档。组织得很好，搜索功能强大，你还可以在上端切换版本。它只是缺少一个关于“最佳实践教程”的精彩内容，这正是本书试图填补的。
    
    * [高效现代 CMake](https://gist.github.com/mbinna/c61dbb39bca0e4fb7d1f73b0d66a4fd1)：一份关于该做什么和不该做什么的极好清单。
    
    * [拥抱现代 CMake](https://steveire.wordpress.com/2017/11/05/embracing-modern-cmake/)：一篇对术语有良好描述的文章。
    
    * [是时候正确使用 CMake 了](https://pabloariasal.github.io/2018/02/19/its-time-to-do-cmake-right/) ：一套针对现代 CMake 项目的优秀最佳实践。
    
    * [《现代 CMake 终极指南》](https://rix0r.nl/blog/2015/08/13/cmake-guide/)：一篇略过时的文章，但意图相似。
    
    * [更多现代 CMake](https://youtu.be/y7ndUhdQuU8)：2018 年 Meeting C++上的一次精彩演讲，推荐使用 CMake 3.12+。这次演讲将 CMake 3.0+称为“现代 CMake”，CMake 3.12+称为“更现代的 CMake”。
    
    * [Oh No! 更多现代 CMake](https://www.youtube.com/watch?v=y9kSr5enrSk)：更多现代 CMake 的续集。
    
    * [toeb/moderncmake](https://github.com/toeb/moderncmake)：关于 CMake 3.5+的精彩演讲和示例，通过项目组织介绍语法。

*  [HSF CMake 培训 more modern cmake](https://hsf-training.github.io/hsf-training-cmake-webpage/01-intro/index.html)
