# c++关键字

1. `volatile`

    什么时候用 volatile 或 std::atomic？

    | 场景	               | 使用 volatile	 |使用 std::atomic|
    |:---|:---|:---|
    |硬件寄存器的直接访问	|是	              |否     |
    |信号处理函数中的标志变量|是	          |否     |
    |多线程计数器、标志位	|否	             |是     |
    |控制并发的简单同步（如标志、计数器）|否 |是     |
    |确保多线程操作的原子性	|否	             |是     |
    * 总结
volatile 在信号处理程序中主要用于防止编译器优化，**确保**变量的最新值对主程序和信号处理程序都**可见**。
它提供了一种轻量级的方式来实现基本的同步，但它不解决线程安全问题。
对于更复杂的同步需求，例如多线程环境，应该使用 std::atomic 或其他线程安全机制。

2. `constexpr`
* constexpr声明说明符
    * 指定变量、函数的值可以在常量表达式中出现
    * **constexpr变量**： 1该条声明是定义 2它具有字面类型 3它被（这条声明）初始化。
    * **constexpr函数**：函数是在使用需要它的代码时，可在编译时计算其返回值的函数。当其参数是 constexpr 值时，constexpr 函数会生成编译时常量。 使用非 constexpr 自变量调用时，或者编译时不需要其值时，它将与正则函数一样，在运行时生成一个值。
* constexpr的lambda声明符c++17
    * 显式指定 operator() 为 constexpr 函数。如果 operator()满足针对 constexpr 函数的所有要求，那么即使没有指定 constexpr，operator() 也会是 constexpr 的。

* (备注)**常量表达式**
定义能在编译时求值的表达式。这种表达式能用作**非类型模板实参、数组大小**，并用于**其他要求常量表达式**的语境，例如：
    ```
    int n = 1;
    std::array<int, n> a1;  // 错误：“n” 不是常量表达式
    const int cn = 2;
    std::array<int, cn> a2; // OK：“cn” 是常量表达式
    ```

3. `const`
* const 类型限定符: 定义类型为常量类型。
* const 限定的成员函数: 有 const 限定符的成员函数中只能正常地调用其他有 const 限定符的成员函数。

4. `const_cast`
    * const_cast 只能进行下列转换：
    * 对于两个相似的对象指针或数据成员指针类型 T1 和 T2，如果 T1 和 T2 仅在 cv 限定上有不同
    * 对于两个对象类型 T1 和 T2，如果指向 T1 的指针可以通过 const_cast<T2*> 显式转换到类型“指向 T2 的指针”，那么也可以进行以下转换：
        * T1 类型的**左值**可以通过 const_cast<T2&> 显式转换成类型 T2 的**左值**。
        * T1 类型的**泛左值**可以通过 const_cast<T2&&> 显式转换成类型 T2 的**亡值**。
        * 如果 T1 是类类型或数组类型，那么 T1类型的**纯右值**可以通过 const_cast<T2&&> 显式转换成类型 T2 的**亡值**。
    * **移除常量性**
    只有 const_cast 能用来移除常量性。“移除常量性”也意味着“移除易变性”，因为限定性转换同样无法移除易变性。函数指针和成员函数指针无法用于 const_cast。const_cast 使得到非 const 类型的引用或指针能够实际**指代** const 对象，或到非 volatile 类型的引用或指针能够实际**指代** volatile 对象。通过非 const 访问路径**修改** const 对象和通过非 volatile 泛左值涉指 volatile 对象是未定义行为。

5. `decltype` 
* 包含引用和cv auto则不
* decltype 说明符 <https://zh.cppreference.com/w/cpp/language/decltype>
    * decltype (实体) : 没有括号的标识表达式或没有括号的类成员访问表达式，那么 decltype 产生该表达式指名的实体的类型。
    * decltype (表达式) : 
        * a.亡值，则 decltype 产生 T&&
        * b.左值，则 decltype 产生 T&
        * c.纯右值，则 decltype 产生 T

* 注意如果对象的名字**带有括号**，那么它会被当做普通的**左值表达式**，因此 decltype(x) 和 decltype((x)) 通常是不同的类型。在所声明的类型难以或不可能以标准写法声明时，decltype 很有用，例如 lambda 相关类型或依赖于模板形参的类型。
    ```
    struct A { double x; };
    const A* a;
    decltype(a->x) y;       // y 的类型是 double（其声明类型）
    decltype((a->x)) z = y; // z 的类型是 const double&（左值表达式）
    ```
* decltype(auto)(C++14 起)占位符 auto 可伴随如 const 或 & 这样的修饰符，它们参与类型推导。

7. `explicit`
* 指定**构造函数(单参数)或转换函数(重载operator type)**(C++11 起)或模版实参推导(C++17 起)为显式，即它不能用于隐式转换和复制初始化。

8. register
* register 用来提示声明的变量会频繁使用，因此它的值可以存储在 CPU 寄存器中。此提示可以忽略，并且大多数实现在需要取声明的变量的地址的情况下都会忽略此提示。这种用法已弃用(c++17)。

9. `extern`
* 具有外部链接的静态存储期说明符
* 语言链接说明 extern "c++"(默认)/"c"
    * extern "c"既可以在c++里使用c函数，也可以使得c单元使用c++的函数。

10. `friend`
* 友元声明在类体内出现，并向一个**函数或另一个类**授予对包含友元声明的类的**私有及受保护**成员的访问权。

11. `inline`
* 在函数的 声明说明符序列 中使用时，将函数声明为一个 内联函数。整个定义都在 class/struct/union 的定义内的函数，都是隐式的内联函数，无论它是成员函数还是非成员 friend 函数。
*inline namespace 命名空间名 { 声明序列 } 命名空间 命名空间名 的内联命名空间定义。命名空间名 内的声明将在它的外围命名空间可见。

12. `mutable`
* mutable 类型说明符: 允许常量**类类型对象**修改相应类成员（即类**成员可变**）。
* mutable 用于指定不影响类的外部可观察状态的成员（通常用于互斥量、记忆缓存、惰性求值和访问指令等）。(类的无关属性，辅助额外的数据)
```
class ThreadsafeCounter
{
    mutable std::mutex m; // “M&M 规则”：mutable 与 mutex 一起出现
    int data = 0;
public:
    int get() const
    {
        std::lock_guard<std::mutex> lk(m);
        return data;
    }
    void inc()
    {
        std::lock_guard<std::mutex> lk(m);
        ++data;
    }
};
```

* 从按复制捕获的形参中移除 const 限定性的 lambda 声明符(C++11 起): 允许函数体**修改复制捕获的对象**，以及**调用它们的非const成员函数**。

13. noexcept
* noexcept 运算符: noexcept 运算符进行编译时检查，在表达式声明不会抛出任何异常的情况下返回 true。
* noexcept 说明符: 当函数声明为 noexcept 时，它明确表明该函数在运行时不会抛出任何异常。不会抛出的函数允许调用有可能会抛出的函数。每当抛出异常且对处理块的查找遇到了不会抛出的函数的最外层块时，就调用函数 std::terminate。
    * 适用场景
        * 性能关键代码：如标准容器的移动构造和赋值操作。
        * 资源管理函数：如析构函数通常是 noexcept，以确保资源释放时不会有意外异常。
        * 函数库设计：对外发布的接口，使用 noexcept 明确函数行为，提升代码可预测性和鲁棒性。

14. `static_assert` （编译时断言）
* 带有固定错误信息的静态断言: `static_assert( 布尔常量表达式 , 不求值字符串 )`	
* 不带错误信息的静态断言。`static_assert( 布尔常量表达式 )`

15. `thread_local`
* 声明有 thread_local 的变量都具有线程存储期。实体的存储在创建它们的线程的持续期间持续存在。每个线程都有一个不同的对象或引用，并且使用声明的名字时会指代与当前线程关联的实体。

16. virturl 
* 虚函数说明符 : 指定非静态成员函数为虚函数并支持多态。它只能在非静态成员函数的首个声明（即当它在类定义中声明时）的声明说明符序列中出现。
    * 纯虚函数是它的声明符拥有下列语法的虚函数：声明符 虚说明符(可选) = 0; ()
    * 抽象类（abstract class） 是这样的类：不能被实例化，但可以用作基类的抽象类型。它所定义或继承的函数中至少一个的最终覆盖函数是纯虚函数。
* 虚基类说明符 ： virtual [访问说明符] 类或计算类型; 解决菱形继承问题。

* 初始化顺序 1.虚基类按声明顺序 2.直接基类按声明顺序 3.非静态成员按声明顺序 4.构造函数
* 析构顺序：初始化的逆序

17. final override
* final: 指示某个虚函数不能在派生类中被覆盖，或者某个类不能被派生。
```
struct b0 final {};
struct d0: b0 {}; // 错误：不能从 final 基类派生
 
struct b1
{
    virtual void f0() final;
    virtual void f1();
};
 
struct d1: b1 // OK
{
    void f0(); // 错误：不能覆盖 final 函数
    void f1() override; // OK
};
```
