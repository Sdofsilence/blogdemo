#### cppreference零三五规则

1. 三原则 如果用户需要**析构、复制构造、复制赋值函数**之一，那么它几乎肯定需要所有的三个。
2. 五原则 用户定义(包括default和delete)的**析构函数、复制构造、复制赋值函数**，会阻止隐式的**移动构造和移动赋值运算符**，所以任何想要移动语义的类必须声明所有五个特殊成员函数。
3. 零原则 如果可以避免定义默认函数，就不要定义它们(析构函数、复制构造、复制赋值函数、移动构造、移动赋值运算符)。多态基类析构函数需要public 和 virtual，阻止了拷贝和移动。多态类如果没有数据，应该delete copy/move(四个),有数据，把它们设置成protected。

#### 隐式声明规则

1. 没有用户声明的构造函数或构造函数模板，那么编译器会隐式声明一个 inline public 成员默认构造函数。
2. 没有用户声明的析构函数，那么编译器总是会声明一个 inline public 成员析构函数。
3. 没有用户定义的复制构造函数，那么编译器总是会声明一个作为这个类的非 explicit 的 inline public 成员复制构造函数。
4. 没有用户定义的复制赋值运算符，那么编译器将始终声明一个复制赋值运算符作为类的 inline public 成员。
5. 没有用户定义的移动构造函数，
    * 没有用户声明的复制构造函数；
    * 没有用户声明的复制赋值运算符；
    * 没有用户声明的移动赋值运算符；
    * 没有用户声明的析构函数；

    那么编译器将声明一个非 explicit 的 inline public 成员移动构造函数。
6. 没有用户定义的移动赋值运算符，
    * 没有用户声明的复制构造函数；
    * 没有用户声明的复制赋值运算符；
    * 没有用户声明的移动构造函数；
    * 没有用户声明的析构函数；

    那么编译器将声明一个非 explicit 的 inline public 成员移动赋值运算符。

#### 隐式弃置规则

1. 类 T 中隐式声明的复制构造函数在 T 声明了移动构造函数或移动赋值运算符的情况下会被弃置。
2. 类 T 中隐式声明的复制赋值运算符在 T 声明了移动构造函数或移动赋值运算符的情况下会被弃置。

#### C++11 标准文档（ISO/IEC 14882:2011）中关于复制构造函数和复制赋值运算符的隐式弃置(=delete)的说明(cppreference中非平凡的)

1. 如果成员或基类不能拷贝 → 该类的默认拷贝赋值运算符和拷贝构造函数被隐式删除。
2. 如果成员或基类不能移动 → 该类的默认移动赋值运算符和移动构造函数被隐式删除。
3. 具体哪些成员或者基类不能拷贝或者移动见c++11标准文档(或用ai问)

### 微软的说明

* Move operations are only generated when no destructor or move or copy operations are declared.
* A default copy constructor is only generated when no copy constructor is declared. It is implicitly deleted if a move operation is declared. 
* A default copy assignment operator is generated only when no copy assignment operator is explicitly declared. It is implicitly deleted if a move operation is declared.
