# c++表达式类别 
* c++每个表达式都具有某种非引用类型，且每个表达式只属于三种基本值类别中的一种：纯右值 (prvalue)、亡值 (xvalue)、左值 (lvalue)。

## 精简
* 非引用+临时量：纯右值
* 左值引用+标识符:左值
* 亡值：右值引用+return表达式

## 泛左值：左值+亡值 | 右值：亡值+纯右值

* **泛左值**:泛左值可以通过左值到右值、数组到指针或函数到指针隐式转换转换成纯右值。

* **左值**: 
    1. 可取地址&
    2. 可作为复值运算符的左操作数
    3. 初始化左值引用

    一般的：返回左值引用的表达式static_cast<int&>(x);

    特别记忆：1.字符串字面量 2.函数的右值引用转型static_cast<void(&&)(void)(x)>

* **右值**:包括纯右值和亡值
    1. 不能取地址：&int()、&i++[3]、&42 及 &std::move(x) 是非法的。
    2. 不能用作赋值运算符左操作数。
    3. 可以用来初始化 const 左值引用，这种情况下该右值所标识的对象的生存期被延长到该引用的作用域结尾。
    4. 右值可以用来初始化右值引用，这种情况下该右值所标识的对象的生存期被延长到该引用的作用域结尾。

* **纯右值**：任何完整类型 T 的纯右值，可转换成同类型 T 的亡值。
    1. 除了字符串字面量之外的）字面量，例如 42、true 或 nullptr
    2. 返回类型是非引用的函数调用或重载运算符表达式
    3. 转换到非引用类型的转型表达式，例如 static_cast<double>(x)、std::string{} 或 (int)42
    4. 其他表达式能代表一个计算结果或表示一个值但是没有标识符的临时量。例如：a++,a+b,a&&b,a<b,&a,this指针

* **亡值**:性质同 泛左值 和 右值。
    1. 返回类型是对象的右值引用的函数调用或重载运算符表达式，例如 std::move(x)
    2. 转换到对象的右值引用类型的类型转换表达式，例如 static_cast<char&&>(x)
    3. 在临时量实质化后，任何指代该临时对象的表达式。(C++17 起)(绑定引用到纯右值时)
    4. 有移动资格的表达式。

## 另一种解释

### C++11 中：

* 拥有身份且不可被移动的表达式被称作左值 (lvalue) 表达式
* 拥有身份且可被移动的表达式被称作亡值 (xvalue) 表达式
* 不拥有身份且可被移动的表达式被称作纯右值 (prvalue) 表达式
* 不拥有身份且不可被移动的表达式无法使用。

拥有身份的表达式被称作“泛左值 (glvalue) 表达式”。左值和亡值都是泛左值表达式。

可被移动的表达式被称作“右值 (rvalue) 表达式”。纯右值和亡值都是右值表达式。

### C++17
C++17 中，某些场合强制要求进行复制消除，而这要求将纯右值表达式从被它们所初始化的临时对象中分离出来，这就是我们现有的系统。要注意，与 C++11 的方案相比，`纯右值已不再是可被移动`。

## 有移动资格的表达式

虽然由任何变量的名字构成的表达式是左值表达式，但若它作为

* return 语句
* co_return 语句 (C++20 起)
* throw 表达式 (C++17 起) 

的操作数出现，则表达式具有移动资格。
