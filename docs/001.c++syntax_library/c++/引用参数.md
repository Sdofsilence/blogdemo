# c++引用参数

### 函数

```
void fun7(const int&) {
    cout << "const int& \n";
}

void fun7(int&) {
    cout << "int& \n";
}

void fun7(int&&) {
    cout << "int&& \n";
}

void fun7(const int&&) {
    cout << "const int&& \n";
}

int main(void) {
    int a = 1;
    fun7(2);
}
```
* 左值fun(int&) 次选fun(const int&)
* 右值fun(int&&) 次选fun(const int&&)
* const左值fun(const int&)
* const右值fun(const int&&)

* 总结：1.cv限定的可以接受无cv的参数 2.const & 可以接受左值和右值和常量(支持所有参数类型)

### 模版转发问题引用折叠问题

#### 转发引用

转发引用是一种特殊的引用，它保持函数实参的值类别，使得 std::forward 能用来转发实参。转发引用是下列之一：

1. 函数模板的函数形参，其被声明为同一函数模板的类型模板形参的无 cv 限定的右值引用：
```
template<class T>
int f(T&& x)                      // x 是转发引用
{
    return g(std::forward<T>(x)); // 从而能被转发
}
```
2. auto&&，但当其从花括号包围的初始化器列表推导时(auto&& z = {1, 2, 3};)
```
auto&& vec = foo();       // foo() 可以是左值或右值，vec 是转发引用
auto i = std::begin(vec); // 也可以
(*i)++;                   // 也可以
 
g(std::forward<decltype(vec)>(vec)); // 转发，保持值类别
 
for (auto&& x: f())
{
    // x 是转发引用；这是在泛型代码中安全使用范围 for 循环的一般方法
}
```

#### 引用折叠
* 通过模板或 typedef 中的类型操作可以构成引用的引用，此时适用引用折叠 ﻿规则：右值引用的右值引用折叠成右值引用，所有其他组合均折叠成左值引用：
```
typedef int&  lref;
typedef int&& rref;
int n;
 
lref&  r1 = n; // r1 的类型是 int&
lref&& r2 = n; // r2 的类型是 int&
rref&  r3 = n; // r3 的类型是 int&
rref&& r4 = 1; // r4 的类型是 int&&
```
（这条规则，和将 T&& 用于函数模板时的模板实参推导的特殊规则一起，组成了使得 std::forward 可行的规则。）即对于转发引用，左值推导为左值引用，右值推导为右值引用。(以下是摘录自cppreference的描述)

> 3) 如果 P 是引用类型，那么用 P 所引用的类型推导。
> 4) 如果 P 是到无 cv 限定模板形参的右值引用（也就是转发引用）且对应函数的调用实参是左值，那么将到 A 的左值引用类型用于 A 的位置进行推导（注意：这是 std::forward 的行动基础。
