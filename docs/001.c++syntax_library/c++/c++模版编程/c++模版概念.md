# 模版概论
## 分类(模板是以下实体之一)
* 类模版(嵌套类)
* 函数模版(成员函数)
* 模版别名 c++11
* 变量模版 c++14
* 约束和概念 c++20

## 模板以一个或多个模板形参参数化，形参有三种(及它们的形参包)：
* 类型模板形参
* 非类型模板形参和
* 模板模板形参
    ~~~
    template<typename T>
    class my_array {};
    // 类型模板形参、非类型模版形参、模板模板形参：
    template<typename K, int count, template<typename> typename C = my_array>
    class Map {
        C<K> key;
        C<V> value;
    };
    ~~~

## 特化
* 当提供了模板实参，或当函数模板或类(C++17 起)模板的模板实参被推导出时，它们替换各模板形参，以获得模板的一个特化，即一个特定类型或一个特定函数左值。
* ###特化也可以显式提供：对类、变量(C++14 起)和函数模板都允许全特化，只允许对类模板和变量模板(C++14 起)部分特化。###
* 在要求*完整对象类型*的语境中引用某个*类模板特化*，或在要求*函数定义存在*的语境中引用某个*函数模板特化*，模板即被*实例化*（它的代码被实际编译），除非模板已经被*显式特化或显式实例化*。
* 模板的定义必须在*隐式实例化点*可见，这就是模板库通常都在头文件中提供所有模板定义的原因。

## 模板化实体
* **模板化**实体是在模板定义内定义（或对于lambda表达式为创建）(C++11 起)的实体。它包含1.类/函数/变量(C++14 起)模板
2.概念(C++20 起)3.模板化实体的成员（例如类模板的非模板成员函数）4.作为模板化实体的枚举的枚举项5.任何模板化实体中定义或创建的实体：局部类，局部变量，友元函数，等等6.模板化实体的声明中出现的 lambda 表达式的闭包类型(C++11 起)。例如，在以下模板中：
    ~~~
    template<typename T>
    struct A
    {
        void f() {}
    };
    ~~~

* 函数 A::f 不是函数模板，但它仍然会被当做是模板化的。模板化函数是函数模板或模板化的函数。模板化类是类模板或模板化的类。模板化变量是变量模板或模板化的变量。(C++14 起)

## 类模版
### 声明
> template<形参> [约束] 类声明;

### 实例化
#### >显示实例化
> template class 类模版名<实参> 显示实例化定义;
* 显式实例化定义强制实例化其所指代的类型。对于给定的实参列表，它在整个程序中只能出现一次。
> extern template class 类模版名<实参> 显示实例化声明;
* 显式实例化声明（extern 模板）跳过隐式的实例化：本来导致隐式实例化的代码会以别处提供的显式实例化定义取代（如实例化不存在，则连接错误）。此机制用于减少编译时间：只在一个文件中明确定义它，其他的所有源文件中显示模板实例化声明。
* 类、函数、变量(C++14 起)和成员模板的特化可以从其模板显式实例化。成员类、成员函数和类模板的静态数据成员可以从其成员定义显式实例化。
#### >隐式实例化
* 当要求完整定义的类型语境中使用某个模板，或当类型的完整性对代码有影响，而特定类型尚未被显式实例化时，发生隐式实例化。
* 适用于**类模板的成员**：除非成员在程序中**被使用**，否则并不会**实例化**它，并且不要求其有定义。
* 如果类模板已声明但未定义，它的实例化会在实例化点产生不完整的类类型。
* 局部类和它的成员中所使用的任何模板都会作为该局部类或枚举所处于的实体的实例化的一部分进行实例化。(C++17 起)
## 函数模版
### 声明
> template<形参> [约束] 函数声明;
* c++20可用[概念] auto简写函数模版
### 实例化
#### 显示实例化
> [extern] template 返回类型 函数名<类型实参>(形参); 显式指定所有无默认值的模板形参时，就不会触发模板实参推导
> [extern] template 返回类型 函数名(形参); 对所有形参进行模板实参推导
* 以extern 区分显示实例化定义 和 显示实例化声明
* 在函数模板特化或成员函数模板特化的显式实例化中，尾部的各模板实参在能从函数参数推导时不需要指定
#### 隐式实例化
* 在要求存在函数定义的语境中指涉某个函数，或定义存在与否会影响程序语义(C++11 起)，而这个特定函数尚未被显式实例化时，发生隐式实例化。如果模板实参列表能从语境推导，那么不必提供它。

**注意**：完全省略 <> 允许重载决议同时检验模板与非模板重载。

### 模版实参推导
* 实例化一个函数模板需要知道它的所有模板实参，但不需要指定每个模板实参。编译器会尽可能从函数实参推导缺失的模板实参。
* 模板运算符依赖此机制，因为除了将它重写为函数调用表达式之外，不存在为运算符指定模板实参的语法。

### 显式模板实参
* 函数模板的模板实参可从以下途径获得：
* 模板实参推导 
* 默认模板实参
* 显式指定

### 函数模版重载
* 函数模板与非模板函数可以重载。
* 非模板函数与具有相同类型的模板特化始终不同。
* 不同函数模板的特化，即使具有相同类型，也始终互不相同。
* 两个具有相同返回类型和相同形参列表的函数模板是不同的，而且可以用显式模板实参列表进行区分。
* 当使用了类型或非类型模板形参的表达式在函数形参列表或返回类型中出现时，该表达式会为了重载而保留为函数模板签名的一部分
* 对于两个涉及模板形参的表达式，如果两个包含这些表达式的函数定义根据 单一定义规则相同，那么称它们等价，就是说，除了模板形参的命名可以不同之外，这两个表达式含有相同的记号序列，其中的各个名字通过名字查找都解析到相同的实体。
    ~~~
    template<int I, int J>
    A<I+J> f(A<I>, A<J>); // 重载 #1

    template<int K, int L>
    A<K+L> f(A<K>, A<L>); // 同 #1

    template<int I, int J>
    A<I-J> f(A<I>, A<J>); // 重载 #2
    ~~~

### 重载偏序处理
* 当同一个函数模板特化与多于一个重载的函数模板相匹配时（这通常由模板实参推导所导致），执行重载函数模板的偏序处理以选择最佳匹配。在以下情形中发生偏序处理：
* > template<class X>\
  > void f(X a);\
  > template<class X>\
  > void f(X* a);

* 1.对函数模板特化的调用的重载决议
  > int* p; f(p);
* 2.取函数模板特化的地址时
  > void f(X* a); void (*p)(int*) = &f;
* 3.选择作为函数模板特化的布置 operator delete 以匹配布置 operator new 时
* 4.当友元函数声明、显式实例化或显式特化指代函数模板特化时
  > template<> void f<>(int *a) {} // 显式特化 \
  > // 模板实参推导出现两个候选：f<int*>(int*) 与 f<int>(int*) 偏序选择 f<int>(int*)，因为它更特殊

* 非正式而言，“A 比 B 更特殊”意味着“A 比 B 接受更少的类型”。

### 函数重载 vs 函数特化
注意，只有非模板和主模板重载参与重载决议。特化并不是重载，因此不受考虑。只有在重载决议选择最佳匹配的主函数模板后，才检验它的特化以查看最佳匹配者。在对翻译单元的头文件进行排序时，记住此规则很重要,更多示例<https://zh.cppreference.com/w/cpp/language/function_template>。

## 约束和概念c++20
* 类模板，函数模板（包括泛型 lambda），以及其他模板化函数（通常为类模板的成员），可以与一项**约束**相关联，它指定了对模板实参的一些要求，这些要求可以被用于选择最恰当的函数重载和模板特化。
* 这种要求的具名集合被称为**概念**。每个概念都是一个谓词，它在编译时求值，并在将之用作约束时成为模板接口的一部分。


### 约束与约束表达式
#### 约束
* 约束是逻辑操作和操作数的序列，它指定了对模板实参的要求。它们可以在 requires 表达式中出现，也可以直接作为概念的主体。
* 有三种约束：1.合取 2.析取 3.原子 4.折叠c++26
#### requires约束表达式
* 产生描述约束的 bool 类型的纯右值表达式。
> requires [(形参)] {要求序列};
#### 要求序列：
* 简单要求 `表达式;`  简单要求断言非requires开头表达式是有效的。表达式是不求值操作数。
* 类型要求 `typename 标识符 ;`  类型要求断言标识符指名的类型是有效的：可以用来验证指名的嵌套类型是否存在，或者某个类/别名模板特化是否指名了某个类型。
* 复合要求 `{表达式} [noexcept] [->类型约束]`复合要求断言表达式的属性。
* 嵌套要求 `requires 约束表达式` 根据本地形参指定其他约束。

### 概念
* 概念是要求的具名集合。
> template<形参> <br> concept 概念名 = 约束表达式;
* 概念不能递归不能被约束。不能更改约束的原初定义。
* 概念在 类型约束 中接受的实参要比它的形参列表要求的要少一个，因为按语境推导出的类型会隐式地用作第一个实参：
    ~~~
    template<class T, class U>
    concept Derived = std::is_base_of<U, T>::value;
    
    template<Derived<Base> T>
    void f(T); // T 被 Derived<T, Base> 约束
    ~~~
