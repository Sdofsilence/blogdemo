# 基于维基分类
## 执行器强化
1. 右值引用和move语义
2. constexpr – 泛化的常量表达式
3. 对POD定义的修正 C++11通过把POD概念划分成两个概念：平凡的（trivial）和标准布局（standard-layout），放宽了关于POD的定义。**一个平凡的类型可以被静态初始化，意味着使用memcpy来复制数据是合法的**，而无须使用复制构造函数。平凡的类型对象的生命周期开始于其存储空间被分配时，而不是其构造函数完成时。使用模版类std::is_trivial<T>::value来判断数据类型是否为平凡类型。**一个符合标准布局的类封装成员的方式与C兼容**。一个类、结构、联合只有在其是平凡的、符合标准布局，并且所有非静态成员和基类都是POD时，才被视为POD。使用<type_traits>中的is_pod<T>::value判断T是不是POD类型

## 易用性 
* 为了使C++能够更容易使用。举凡可以增进类别安全，减少代码重复，不易误用代码之类的。
1. 统一/列表 初始化(initializer_list做参数) 带=号是复制初始化，只考虑转换构造函数(不以说明符 explicit 声明的构造函数+非显式的用户定义转换函数)<https://zh.cppreference.com/w/cpp/language/converting_constructor>
2. 类型推导auto/decltype decltype带cv和引用&/&&
3. range-for  `for(auto &i ：rangeable)`任何定义了begin()和end()的类型。
4. lambda表达式(匿名函数) 定义在与lambda函数相同作用域的参数引用也可以被使用。这种的参数集合一般被称作closure（闭包）。
5. 返回类型后置声明 `auto FuncName(int x, int y) -> int;`关键字auto的使用与其在自动类别推导代表不同的意义。
6. 构造函数的改良(委托和继承构造函数) 允许构造函数调用其他构造函数，这种做法称作委托（delegation）构造。C++11允许派生类手动继承基类别的构造函数，编译器可以使用基类别的构造函数完成派生类的构造。而将基类的构造函数带入派生类的动作，无法选择性地部分带入，要不就是继承基类全部的构造函数，要不就是一个都不继承（不手动带入）。

```
    class BaseClass
    {
    public:
      BaseClass(int iValue);
    };

    class DerivedClass : public BaseClass
    {
    public:
      using BaseClass::BaseClass;
    };
```

7. final/override  防止重载和显式重载 不是关键字??只有在特定的地方有特别含义
8. nullptr C++11引入了新的关键字来代表空指针常量：nullptr，将空指针和整数0的概念拆开。 nullptr的类别为nullptr_t，能隐式转换为任何指针或是成员指针的类别，也能和它们进行相等或不等的比较。而nullptr不能隐式转换为整数，也不能和整数做比较。
9. 强类型有作用域枚举enum class/struct 此种枚举为类别安全的。枚举类别不能隐式地转换为整数；也无法与整数数值做比较。在C++11中，枚举类别的前置声明（forward declaration）也是可行的，只要使用可指定类别的新式枚举即可。之前的C++无法写出枚举的前置声明，是由于无法确定枚举参数所占的空间大小，C++11解决了这个问题
10. 显式类别转换explicit C++98引入了关键字explicit来避免用户自定的单形参构造函数被当成隐式类别转换子。在C++11中，关键字explicit修饰符也能套用到类别转换函数上。如同构造函数一样，它能避免类别转换函数被隐式转换调用。但C++11特别指定，在if条件式、循环、逻辑运算等需要布尔值的地方，将其作为显式类型转换，因此即使对应的类型转换函数被explicit修饰也可以调用。这主要为了解决safe bool问题。
11. using别名 `template< typename second> using TypedefName = SomeType<OtherType, second, 5>; using PFD = void (*)(double);`
12. 模版参数缺省值 C++11可以支持函数模板的模板参数默认值。类模板的模板参数默认值需要从右到左依次出现；函数模板的默认模板参数没有此约束。
13. 无限制的unions 移除了所有对union的使用限制(可以有非平凡的构造函数的类型必须自定义)，除了其成员仍然不能是引用类别。 这一改变仅放宽union的使用限制，不会影响既有的旧代码。

```
    union U {
    int z;
    Point /*struct*/ p;  // 在C++03中是不合法（point有一non-trivial建構式），但是在C++11是合法的
    U() {} // 由于 Point 成员的存在，必须要定义一个构造函数};
```

## 新功能
1. 可变参数模板 长参数模板中，变长参数包无法如同一般参数在类或函数中使用； 因此典型的手法是以递归的方法取出可用参数。
完整的参数可以使用包展开。后随省略号且其中至少有一个形参包的名字的模式会被展开 ﻿成零个或更多个逗号分隔的模式实例，其中形参包的名字按顺序被替换成包中的各个元素。

```
    #include <iostream>
    void tprintf(const char* format) // 基础函数 {
        std::cout << format;
    } 
    template<typename T, typename... Targs>
    void tprintf(const char* format, T value, Targs... Fargs) // 递归变参函数 {
        for (; *format != '\0'; format++) {
            if (*format == '%') {
                std::cout << value;
                tprintf(format + 1, Fargs...); // 递归调用
                return;
            }
            std::cout << *format;
        }
    }
    int main() {
        tprintf("% world% %\n", "Hello", '!', 123);
    }
```

2. 新字符串字面量 C++11将支持三种Unicode编码方式：UTF-8，UTF-16，和UTF-32。字符串字面量：u8"utf-8"),u"utf-16",U"utf-32",新增char16_t,char32_t两种类型。三种字符串的类型分别是，const char[],const char16_t[],const char32_t[]。C++11将提供raw（原始）字符串字面值：R"raw string"其中"和\字符不需要经过转义。

3. 用户定义字面量 C++11开放用户定义新的字面修饰符（literal modifier），利用自定义的修饰符完成由字面值构造对象。

4. thread-local线程存储期 多线程环境下让各线程拥有各自的参数是很普遍的。这已经存在于函数的区域参数，但是对于**全局和静态参数**都还不行。线程区域的存储期限会借由存储指定字thread_local来表明。

5. default,delete 使用或禁用默认函数

6. long long int 保有至少64个有效比特的整数类别

7. static_assert 静态assertion C++提供了两种方法测试assertion（声明）：宏assert以及预处理器指令#error。但是这两者对于模版来说都不合用。宏在执行期测试assertion，而预处理器指令则在前置处理时测试assertion，这时候模版还未能实例化。使用新的关键字static_assert,可以在编译期测试assertion。

8. 允许sizeof运算符作用在类别的数据成员上

9. 垃圾回收机制 void declare_reachable(void* p);
template<class T> T* undeclare_reachable(T *p) noexcept;
void declare_no_pointers(char*p ,size_t n)noexcept;
void undeclare_no_pointers(char*p ,size_t n) noexcept;

## 标准库
1. 标准库组件上的升级 目前的标准库能受益于C++11新增的一些语言特性。
2. 线程支持 thread mutex condition_variavle packaged_task,async,future
3. tuple
4. hash_table unordered_set,unorderde_map头文件
5. regex正则表达式 这个新的程序库，被定义于<regex>头文件，由几个新的类别所组成：正则表达式（模式）以模板类basic_regex的实体表示模式匹配的情况以模板类match_results的实体表示函数regex_search是用来搜索模式；若要搜索并取代，则要使用函数regex_replace，该函数会回传一个新的字符串。算法regex_search和regex_replace接受一个正则表达式（模式）和一个字符串，并将该模式匹配的情况存储在struct match_results。
6. 智能指针 unique_ptr shared_ptr weak_ptr 一个weak_ptr指向的是一个被shared_ptr所指向的对象。该weak_ptr可以用来决定该对象是否已被销毁。weak_ptr不能被解引用;想要访问其内部所保存的指针，只能透过shared_ptr。有两种方法可达成此目的。第一，类别shared_ptr有一个以weak_ptr为参数的构造函数。第二，类别weak_ptr有一个名为lock的成员函数，其回返值为一个shared_ptr。weak_ptr并不拥有它所指向的对象，因此不影响该对象的销毁与否。
7. 可拓展的随机数 random头文件
8. reference_wrapper包装引用 我们可以透过实体化模板类reference_wrapper得到一个包装引用（wrapper reference）。包装引用类似于一般的引用。对于任意对象，我们可以透过模板类ref得到一个包装引用（至于constant reference则可透过cref得到）。当模板函数需要形参的引用而非其拷贝，这时包装引用就能派上用场。
9. funcation多态函数对象包装器（函数指针，成员函数指针或仿函数 lambda）
10. 用于计算函数对象回返类型的统一方法 result_of/invoke_result(c++17起) std::result_of 背后的动机是为了确定调用可调用 (Callable) 类型的结果，尤其是结果类型对不同实参集不同的情况。<https://zh.wikipedia.org/wiki/C%2B%2B11#%E7%94%A8%E4%BA%8E%E8%AE%A1%E7%AE%97%E5%87%BD%E6%95%B0%E5%AF%B9%E8%B1%A1%E8%BF%94%E5%9B%9E%E7%B1%BB%E5%9E%8B%E7%9A%84%E7%BB%9F%E4%B8%80%E6%96%B9%E6%B3%95>
11. iota 函数可将给定区间的值设定为从某值开始的连续值 `std::array<int, 10> ai; std::iota(ai.begin(), ai.end(), 1);`
