# c++17新特性
* 参考维基百科<https://zh.wikipedia.org/wiki/C%2B%2B17>

## 新的语言特性
1. 类模板的模板参数推导 模板类构造函数能自动推导模板参数的类型(Class template argument deduction, CTAD)。例如允许以`pair(5.0, false)` 取代`pair<double,bool>(5.0, false)`
2. 用auto声明非类型模板参数 非类型模板参数情况下，可从实参类型推导出模板参数:`template <auto... seq>`
3. 折叠表达式(函数式编程) 
4. 初始化列表自动推导规则的修改 以前auto x {3}; 将被推导为std::initializer_list<int>, 现在推导为int。
5. constexpr lambda
6. Lambda函数中按值捕获this指针 以前this指针在lambda函数只能按引用捕获。现在 *this 捕获对象的副本，而 this (C++11)继续捕获引用。
7. 内联变量(类内定义) 过去关键字inline可用于函数声明，现在也可以用于变量声明，表示函数或定义可定义（但内容必须完全相同）多次。这允许在头文件中定义一个内联变量。内联变量必须是静态存储器（类静态成员或命名空间作用域变量），不能用于块作用域变量或函数作用域变量。
8. 嵌套的命名空间
9. 结构化绑定 变量定义初始化时，允许形如 auto [ x, y, z ] = expr;，其中expr的“元组类似的对象”包括std::tuple, std::pair, std::array等聚合结构。
    ```
    using Coordinate = std::pair<int, int>;
    Coordinate origin() {
      return Coordinate{0, 0};
    }
    const auto [ x, y ] = origin();
    x; // == 0
    y; // == 0
    std::unordered_map<std::string, int> mapping {
      {"a", 1},
      {"b", 2},
      {"c", 3}
    };
    // Destructure by reference.
    for (const auto& [key, value] : mapping) {
      // Do something with key and value
    }
    ```
10. 带初始化的选择语句 if和switch语句允许如下的变量声明和初始化：if(;) switch(;)
11. constexpr if
12. utf8字面量 `char x = u8'x';`
13. 枚举类型变量直接用大括号初始化
14. fallthrough, nodiscard, maybe_unused特性 C++17 引入了3个特性（attribute）: fallthrough, nodiscard, maybe_unused.//fallthrough指示编译器直通一个switch语句//nodiscard引发一个警告，当具有该特性的函数或类的返回值被忽略//maybe_unused告诉编译器变量或参数可能未被使用
15. __has_include运算符 __has_include运算符用在#if 和 #elif表达式，以检查是否一个头文件或源文件可否包含。

## 新的库特性
1. std::variant 类型安全的union
2. std::optional 可能包含一个值或者可能不包含值
3. std::any 类型安全的包含任何类型的值
4. std::string_view 不拥有字符串的情况可以读取其值
5. std::invoke 用于调用“可调用对象”。
6. apply 把std::tuple参数应用于可调用对象
7. filesystem 可操作文件、目录、路径
8. byte 既不是字符类型，也不是算术类型，实际上是枚举类型
9. maps和sets上更有效率的移动节点 零开销的节点操作
10. 并行算法 许多STL算法，如copy, find 和 sort，支持并行执行政策：seq, par, par_unseq, unseq分别表示"sequentially", "parallel", "parallel unsequenced", "unsequenced".
